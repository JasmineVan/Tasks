The report is designed for application diagnostics based on the event log data. It displays statistics on application, errors, and warnings occurred during operation, number of active users in different periods of time and during scheduled job execution.

To open the report, go to Administration – Support and maintenance, and then click Administrator reports.

This is a statistic information designed for diagnostics of the application and server workload. It is not a tool to evaluate employee's labor productivity, calculate salary, etc.

Report options

  • User activity analysis
  • User activity
  • Event log control
  • Duration of scheduled jobs

User activity analysis

  • The report displays statistics on user activity (work hours, number of connections, number of entered/edited documents/catalogs, number of errors and warnings during work, the maximum number of users working simultaneously). Some statistics items are shown as a chart for display purposes.
  • Settings:
    • Select Users from the list (you can select all users or user groups).
    • Specify Period.

  • Additional settings: Click Settings. Besides general settings, the following settings are available:

    • By using check boxes, enable or disable information display on the number of entered/edited documents and catalogs as a chart.
    • Select a chart type from the list to display information on the number of entered/edited documents and catalogs.

User activity

  • The report shows detailed statistics on activity of a selected user (list of created and edited documents/catalogs/business processes/tasks). Some statistics items are shown as a chart for display purposes. You can use this report as a more detailed view of the User activity analysis report option by a certain user.
  • Settings:
    • Select a User from the list.
    • Specify Period.

  • Additional settings: Click Settings. Besides general settings, the following settings are available:

    • Select required check boxes: choose Yes to Output information on business processes, tasks, catalogs, and documents, or select No not to show it. By default, the report shows information on catalogs and documents.

Event log control

  • The report is a list of critical event log records (records on errors and warnings). It allows you to control troubleshooting problems during application operation. The report helps you quickly analyze the frequent issues and make decisions on troubleshooting them without resource to the event log.
  • Settings: Specify Period.

Duration of scheduled jobs

  • The report shows time of execution and duration of work sessions of scheduled jobs for the selected day. It allows you to analyze server workload with scheduled jobs, detect intervals with the maximum load, and unload them.
  • By double-clicking a scheduled job name in the report, you can run the Scheduled job information report, and view the statistics on the selected scheduled job.
  • Settings:
    • Select a Day to get information for the part of this day.
    • Specify Period start – time of the selected day beginning from which you want to get information on scheduled jobs.

  • Additional settings: Click Settings. Besides general settings, the following settings are available:

    • Specify Period end – time of the selected day till which you want to get information on scheduled jobs.
    • Select Yes to Show background jobs. Clear the check box, or select No not to show them.
    • Specify Minimum length of scheduled job sessions (in seconds). The default value is 1.
    • Specify Minimum concurrent scheduled jobs for period. The default value is 2.
    • Select the check box to Hide scheduled jobs.

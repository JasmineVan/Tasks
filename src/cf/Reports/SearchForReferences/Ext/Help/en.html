The report is designed to provide information on list item usage in the application documents.

Using the application, Users can duplicate items by mistake. This will prevent the application from functioning normally. That is why, duplicates must be deleted. If using such duplicates, there can be references to different copies of the item, which may cause errors, for example, incorrect debiting the account and other errors.

To delete duplicates from the application, delete references to them in documents. The report shows the documents, in which the objects are used.

The report can be accessed from application lists by clicking More - Usage locations.

The report displays the following information:

  • Description, Code of selected list items.
  • Usage locations. Document, its number, and date.
  • Occurrence count.

Generating report

  • Select one or several list items, and then click More - Usage locations. The report will be generated automatically.
  • The report is grouped by selected list items. For each item, you can see a list of application documents that contain references to this item, an a number of occurrences. With that, total number of occurrences is counted for each item, and total amount by report.
  • Double-click a corresponding report cell to go to the list items or a document that contain a reference to these items.

This list is used to store additional reports and data processors in an application.

You can connect and utilize a variety of additional (external) reports and data processors in 1C:Enterprise mode. This extends the application's functionality without making any changes to the application, which can be useful when you need to custom-fit the application to specific purposes without losing the technical support rights.

The additional reports and data processors are created in Designer mode. If you want an additional report or data processor created, you need to contact the developers with this request. Alternately, you can download pre-made reports from the Internet.

To enable additional reports and data processors, use the Print forms, reports and data processors command in the Administration section and select the corresponding check box. This will enable the Additional reports and data processors command. By default, the check box is selected.

The list includes the following columns:

  • Name - full name of a report or data processor.
  • Type - there is several types of additional report or data processors, to be specified by developer. Available fucntionality is governed by the report type.
  • Publication - visibility of additional reports and data processors, color-marked. Additional reports and data processors in debug mode are marked in red and accessible by administrator only. Disabled additional reports and data processors are marked in grey. Additional reports and data processors marked for deletion are disabled and marked both in grey and with a special icon.
  • Version - data processor version number specified by developer.
  • Person responsible - application user assigned a person responsible for this report or data processor. This user (by default, Administrator) is responsible for monitoring the additional report/data processor version changes, updating to latest versions, and more.

Using More - Change form, you can add the following fields:

  • Object name - includes kind and other information. This field is filled in by developer and displayed in the appliction by default.
  • Permission compatibility mode - application version used with the report or data processor.
  • Comment - detailed information on the additional report or data processor.
  • Use option storage - property for additional reports. Takes on the Yes value if the additional repost stores data in the application option storage;
  • Use for list form - takes on Yes if the command calling the additional report or data processor is located in list windows.
  • Use for object form - takes on Yes if the command calling the additional report or data processor is located in list item windows.
  • Information - description of the additional report or data processor provided by developer.
  • File name - name and extension of the additional report or data processor file.
  • Safe mode - takes on Yes if the additional report or data processor is executed in safe mode. As a rule, data processors are to be executed in safe mode. This ensures that they do not contain unsafe program code executed on the server. However, certain data processors may require execution in unsafe mode. In this case, a warning will be displayed. These data processors should only be obtained from trusted sources (vendors) or manually audited before execution. For more information on safe mode, refer to the 1C:Enterprise platform documentation.

Registering additional reports or data processors

  • Click Add from file, select an external report or data processor file on your computer (specify full path to it), and configure its parameters. The data list may vary depending on the purpose of the selected report or data processor. If an invalid file was specified, an error message will be displayed (example: the selected additional data processor cannot be used with the current application version).

Filters

The following filters can be enabled to facilitate your navigation of extensive lists of additional reports and data processors:

  • By the Kind field - select an additional report or data processor kind from the filter list (for example, Report).
  • By the Publication field - select a state to be used in filter (for example, Used).
  • By both Kind and Publication fields - in extensive lists, both filters can be combined.

Loading from file

  • Use the Load from file command to update a data processor in the internal application storage. Specify the file path and click Open.

Saving to file

  • Use the Save to file command to export a data processor or report from storage to a file. Specify the folder and click Save. File name will be generated automatically but can be changed if necessary.

Additional report and data processor kinds

Additional reports and data processors are categorized into the following types:

  • Global (common) reports and data processors - these are used independently from the application data. Commands for working with global reports and data processors are located in the application sections. Global reports and data processors can be used to import data into the application, update or clear the full-text search index, etc. They contain commands that can only be called from command interface sections. The commands are independent of any applied objects. The list of sections where commands can be placed is predefined by the developer, included in additional report or data processor file, and cannot be modified.
  • Assignable additional reports and data processors - these are used with specific application lists or documents, single or multiple. The list of objects to which such a report or data processor can be assigned is predefined by developeraк and embedded into text of the report or data processor. When assigning a report, you can only select targets from this list. This type is further broken down into subtypes:
    • Report - assignable additional report. It can be used to create a report on a specific applied object.
    • Object filling - assignable additional data processor that can be used to automatically fill in or clear single or multiple list items.
    • Print form - assignable additional data processor that can be used to create new print forms for a document.
    • Related object creation - assignable additional data processor that can be used to fill in lists quickly by copying list items.

Creating additional data processor or report

  • Create an additional data processor or report.
  • The additional data processor or report is registered in the infobase using the data it provides. These data is to be returned as structure in function ExternalDataProcessorInfo.

Example of how to implement the ExternalDataProcessorInfo function using the application interface

Function ExternalDataProcessorInfo() Export
RegistrationParameters = AdditionalReportsAndDataProcessors.ExternalDataProcessorInfo("2.2.2.1");
RegistrationParameters.Kind = AdditionalReportsAndDataProcessorsClientServer.DataProcessorKind<...>();
RegistrationParameters.Version= "...";
Command = RegistrationParameters.Commands.Add();
Command.Presentation = NStr("en = '<Command presentation>'");
Command.ID = "<Command name>";
Command.Usage = AdditionalReportsAndDataProcessorsClientServer.CommandType<...>();
Command.ShowNotification = <True/False>;
Return RegistrationParameters;
EndFunction

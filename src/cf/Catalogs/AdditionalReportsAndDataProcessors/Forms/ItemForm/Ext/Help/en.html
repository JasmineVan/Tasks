Registering additional reports and data processors in the application.

Some of the data in additional report or data processor cards is filled in automatically. This data was specified by developers in external data processor or report files and cannot be modified (command list, additional information). Any additional data processor can support one or several commands available in applied object windows or in application sections.

Two types of additional reports and data processors are available:

  • Assignable - intended for use with one or more specific application lists or documents. The list of objects to which an additional report or data processor can be assigned is predefined by the developer and specified in the text of the additional report or data processor. When registering an additional report or data processor, you can only select them from the list.
  • Global or common - contain commands that can only be called from command interface sections. The commands are independent of any applied objects. The list of sections where commands can be placed is predefined by the developer, included in additional report or data processor file, and cannot be modified.

Depending on their purpose and settings, additional reports and data processors can contain multiple tabs. In this case, mandatory commands are located on a separate tab. The Additional information tab which is also mandatory contains information on the additional data processor or report. Names and content of other tabs may vary. Special commands can also be located on additional tabs.

When registering an additional report or data processor, you need to specify its primary parameters:

  • Description - provided by developer and filled in automatically. Can be modified if necessary.
  • Operation mode - attachment mode of additional reports and data processors. Possible values: Safe and Unsafe.

  • Publication radio button - controls report or data processor availability. Can be used to limit access to reports or data processors. You can make a report or data processor available for anyone, available for administrator only, or unavailable. This parameter can take on the following values:

    • Used - data processor or report is available for use. Data processor commands are available for all application users with the Use additional reports and data processors role assigned.
    • Debug mode - data processor or report is only available to users with role Add and edit additional data processors or Full access (displayed in red in the list.) For all other users, these commands are hidden. We recommend selecting this mode when an additional report or data processor requires validations.
    • Disabled - data processor or report is unavailable and cannot be selected from the list. We recommend selecting this mode when you need to make a data processor or report temporarily unavailable while keeping all its settings intact. To make a data processor or report available again, select any other mode.
  • Comment - enter detailed description here.
  • Person responsible - assign a user responsible for monitoring additional report or data processor state, updates, and so on. Default assignee is Administrator.

Additional information

  • Additional information provided by developer is shown:
    • Kind - there are the following kinds of additional reports and data processors:
      • Additional report
      • Report
      • Object filling
      • Print form
      • Creating related objects.
    • File name
    • Version
    • Object name
    • Comment - detailed information on the additional report or data processor.
  • Group - add an additional data processor or report to a group. Select a group from the list. Using groups makes it easier to work with additional reports and data processors.

Loading from file

  • Use the Load from file command to update a data processor in the internal application storage. Kind of the data processor being imported must match the current data processor kind. Specify the file path and click Open.

Saving to file

  • Use the Save to file command to export a data processor or report from storage to a file. Specify the folder and click Save. File name will be generated automatically but can be changed if necessary.

Run mode

  • As a rule, data processors are to be executed in safe mode. This ensures that they do not contain unsafe program code executed on the server.
  • However, certain data processors may require execution in unsafe mode. These data processors should only be obtained from trusted sources (vendors) or manually audited before execution.
  • For more information on safe mode, refer to the 1C:Enterprise platform documentation.

Notes on registering assignable additional reports and data processors

  • For additional reports and data processors intended for work with application lists or documents, you can select objects, for which data processor or report commands will be available.
    • Placement. To select application objects, click the hyperlink that lists the selected objects or displays Undefined if no object is selected. Select objects using check boxes.
    • Show - commands for data processors or reports can be available in:
      • The list of selected objects if the In lists check box is selected.
      • The applied object window if the In catalog and document item forms check box is selected. The check boxes are not mutually exclusive and can be both selected.
  • on the Commands tab, the list of commands is filled by default from the list provided by developer. These commands are added to the specified list or document when the additional report or data processor is attached.

Notes on registering global (common) additional reports and data processors

  • For each report and data processor called from command interface (not attached to a configuration object), you need to specify the interface sections where you can access the data processor commands using the Additional reports and Additional data processors commands.
    • Placement - to select interface sections, click the hyperlink that lists the selected sections or displays Undefined when no section is selected. Select the required sections using check boxes and click OK to confirm.
  • List of commands and their settings can be located on a separate Commands tab. The tab shows:
    • Name - full name of the command.
    • In the Quick access column, click (or press F4) and select the users who can access this command by default. The default value is None. Add users to the list. Click OK to confirm selection and return to the setup procedure.
    • In the Schedule column, for each global report or data processor command executed on server side, click and set a scheduled job for running the command. Commands with the Open form launch option enabled cannot be scheduled.

Creating additional data processor or report

  • Create an additional data processor or report.
  • The additional data processor or report is registered in the infobase using the data it provides. These data is to be returned as structure in function ExternalDataProcessorInfo.

Example of how to implement the ExternalDataProcessorInfo function using the application interface

Function ExternalDataProcessorInfo() Export
RegistrationParameters = AdditionalReportsAndDataProcessors.ExternalDataProcessorInfo("2.2.2.1");
RegistrationParameters.Kind = AdditionalReportsAndDataProcessorsClientServer.DataProcessorKind<...>();
RegistrationParameters.Version= "...";
Command = RegistrationParameters.Commands.Add();
Command.Presentation = NStr("en = '<Command presentation>'");
Command.ID = "<Command name>";
Command.Usage = AdditionalReportsAndDataProcessorsClientServer.CommandType<...>();
Command.ShowNotification = <True/False>;
Return RegistrationParameters;
EndFunction

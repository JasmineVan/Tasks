Use this list to store and view IDs of metadata objects of configuration extensions. The IDs are required for application activity and are not user-editable.

This list is populated automatically during the first application run and each subsequent configuration update, on adding and removing extensions, and on request from the code. It is populated according to extensions metadata and updated automatically during each configuration update (including added, deleted, and renamed metadata objects).

It displays all application objects grouped by type. The Synonym object field is displayed by default. Use the More actions — Change form command to add the Full name and ID fields.

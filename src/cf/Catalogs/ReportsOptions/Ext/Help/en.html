The object is aimed at managing report options and configuring placement in report panels of the application sections.

You can save and use multiple options of the same report in the application. There are predefined report options. You can also generate custom report options by changing settings of the existing ones: generation and appearance (fields content, sorting, groupings, conditional formatting) settings.

You can generate common report options. Common report options are available to all users of the report.

The application sections contain commands for going to report panels defined in these sections. If there are many reports, they are combined according to the section groups. For your convenience, you can configure report panels, for example: hide rarely used report options, add frequently used report options to quick access.

Report options administrators can also configure report panel appearance in the application: a report option can be marked as important or placed to the See also group.

To open the list, in the report panel of the application section, execute the All reports command. Sections and groups structure in this list depends on granted rights and roles. All report options without exceptions are available only to an application administrator.

The application administrator can open the list by executing the Report options command in the Administration - Print forms, reports, and data processors section. Some commands or actions available for administrators may be unavailable in the All reports list. This feature is mentioned in the description of list features.

The list consists of two sections:

  • the left section contains a tree of sections and command interface groups that have report panels.
    • When moving from one tree node to another, the list is automatically filtered by the selected section or group.
    • Use the Including subordinate ones check box to display all reports of a section or a group including report options of subordinate groups.
  • The right section contains a report list.

The list includes the following columns:

  • Name - full name of the report option displayed in lists.
  • Author - full name of the user who is the author of the report option.
  • Description - detailed information on the report option.

To add columns to the list, click More - Customize form (settings will be saved):

  • Available to all users picture:
    • is visible - the report option is available to all users,
    • is not visible - the report option is available only to the author.
  • Technical information:
    • Report name - the name under which the report is registered in the application.
    • Report - full report name (presentation).
    • Report type - there are three report types:
      • Internal
      • Additional
      • External - external report options cannot be placed to a report panel.
    • Option key - a name of a predefined report option or a UUID of a user-defined report option.
    • Names of parameters and filters - a list of report parameters and filters.
    • Field names - a list of report field names.

Search

If the report option list is large, you can use the search to find quickly the required report option.

  • In the Search field, enter the required value and click Find. The search is performed in the selected section (group).

    • You can search for reports using a word (a part of a word) or several words. The search is performed in:

      • report names;
      • descriptions of report options;
      • report option field presentations;
      • presentations of report option filters and parameters;
      • presentations of report option filters and parameters;
      • names of sections and groups of the application and the report panel.

Generating report

  • Select a report option, click Open, specify the report settings, and click Run report.
  • In the All reports list, you can also double-click the required report option.

Configure report options

  • Select a report option, click Edit, and edit the required fields.
  • In the Report options list used by an administrator only, you can also double-click the required report option.

Delete report option

  • To delete a user report option, in the More menu, click Mark for deletion. You can also use the Del key.
  • To Remove deletion mark, execute this command again.
  • This feature is unavailable for predefined report options.

Report option placement to application sections

Report option placement can be changed by the report author or an administrator.

  • You can drag report options from the current section/group to another one. If you hold Ctrl while dragging, the report option will be copied to a new group. The application requests confirmation of the report option placement. Click Yes to confirm.

  • You can use the More - Place to sections command to configure placement for multiple report options simultaneously. To do that, specify sections and groups of the command interface to place the report options.

  • You can also configure the report panel appearance in the application: a report option can be marked as Important or placed to the See also group.

  • In the Report options list used by an administrator only, the Place to sections command is displayed in the list command bar for your convenience.

Reset settings

  • Click More - Reset placement settings to restore initial settings of selected report options: placement to the application sections, availability, and visibility by default. This command can be executed only for predefined report options;
  • Click More - Reset user settings to reset user settings of selected report options: visibility and quick access.

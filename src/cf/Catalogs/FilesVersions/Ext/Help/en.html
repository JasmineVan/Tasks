Used to view and manage versions of a file.

To view the list of all file versions, open its card by clicking F2 or More - Open card in the flies list, and click File versions on the navigation panel.

File versions list is hierarchical. It is represented as a tree for easier use. It makes tracking the file editing story easy. Later versions are displayed as subordinate to the earlier ones. The primary version cannot be deleted from the list.

When finished editing, the program offers to save this file version and add information about it. By default, this option is enabled. We recommend that you save versions as in case of fatal errors leading to file damage, you can revert the file to the previous uncorrupted version. To store versions of a file, select the check mark in the file card or during file loading (folders). You can edit version data (for instance, add comments) later.

The list includes the following columns:

  • Author. A user who added the file version. The file type is displayed in this column as a picture.
  • Version number. Numbers are assigned to versions automatically when you change a file. The latest saved version has the highest number.
  • A Comment is a filed for any additional information about a file version.
  • Creation data. A date and time when the file version was added to the infobase.
  • Size (bytes). A file version size in bytes.

With the More - Change form command, you can add fields, such as:

  • Full file name, which shows a full name of a file,
  • Data storing type, which takes values From the infobase or From volumes,
  • Extension, which is a file format, and some other fields.

Viewing file versions

  • Click View to open the selected file version with an application for viewing and editing. You can also open a file version by double-clicking its row or by the More - View command.

Viewing data on file versions

  • Click Open card to open data on a selected version. You can leave or edit a Commentary in a versions card.

Changing a current file version

  • To change a current active version of a file, select a it from the list and click Make active. The numbering of versions will stay the same. Only the order of versions in the list will change.

Comparing two file versions

  • To compare two file versions, select them in the list and click Show changes. Version comparison is supported for the following file types: HTML, TXT, RTF, DOC, DOCX, and ODT.

Saving file versions to hard drive

  • To save a file version to a hard drive or other device, click More actions - Save as. Specify a directory where to save the file.

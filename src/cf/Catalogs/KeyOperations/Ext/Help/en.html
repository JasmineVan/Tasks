This list is used to store key operation parameters.

Application performance tests are based on the key application parameters. The application features standard tools for performance data collection and analysis in accordance with Apdex (Application Performance Index, international standard for software performance evaluation).

To access the list, execute the Key operations command located in the Service subsystems section.

To evaluate application performance, you need to manually fill in parameters for each key operation (as these parameters are not filled in automatically after installation). A number of predefined key operations are available in the application. More operations can be added to the list if necessary. The list includes common operations supported by the application, such as document posting, report generation, and more. The following information is displayed:

  • Name - full name of an operation;
  • Priority - an integer. No key operations can share the same priority;
  • Target time - operation execution time (in seconds);
  • Minimum level - numeric Apdex values are used, for example, Fair (Apdex > = 0.75).

Application performance tests are based on these key operation parameters.

Entering key operation parameters

  • Select a key operation from the list, click More - Change, and fill in the fields.
  • You can also double-click the item.

Creating key operations

  • Click Create and fill in the mandatory fields.

Automatic selection of response time threshold

  • To fill in the Response time threshold field correctly, you can use automatic selection. Click Selection of response time threshold.

Use this window to set up contact information kinds in application documents and lists.

The application allows you to store and process contact information used in your organization.

You can work with contact information in all application lists in the same manner. All lists provide the same functionality for working with addresses or phone numbers. In the application, you can store contact information in tabular sections of documents and load it. You can edit contact information in tabular sections of documents. You might need it when working with accounting reports in digital format. This is because you will likely need to save an exact copy of data submitted to tax authorities (including addresses and phone numbers of individuals).

The application is distributed with predefined groups and list items. Predefined list groups correspond to specific application lists or documents containing contact information. New groups cannot be added to the list. Items contained in these groups define content of contact information of the related application objects.

In lists and tabular sections of application documents, a variety of predefined contact information kinds is available. Besides, new contact information kinds can be added, if necessary.

We recommend that you set up contact information kinds during the initial infobase population stage. However, you can perform the setup later. Usually, the application administrator performs the setup.

You can open the list by clicking the matching command on the navigation panel in Administration - Common settings - Contact information.

Adding new contact information field to list or document

  • Select a group in the list, and then click Create. Fill in the required fields.

Predefined contact information kinds

  • Unlike the arbitrary contact information fields, you can edit only settings for predefined list items. All other attributes are filled in automatically by default and cannot be edited.

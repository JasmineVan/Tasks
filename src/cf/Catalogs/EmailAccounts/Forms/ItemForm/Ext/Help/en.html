Description of email account properties.

When creating an email account, the account parameters are mostly set up automatically, in accordance with the mail server settings.

When creating a system email account, we recommend that you select the Remember password option so that email messages from this account can be sent without user intervention. However, you are free to choose whether to store any account passwords in the infobase or not.

Main additional data

  • Enter email account Name. The name should reflect the goal and purpose of the account or account owner (example: Support service). This name is how the account will be represented in a variety of lists.
  • Enter account Email address in the following format: User@Server.net.
  • Specify Sender name - account owner name that will be used in your emails. It can be different from the user name registered in the application. For example, user Administrator can be registered with sender name Mr. Johnson.

Use account

  • Specify account usage options:
    • Select the Sending messages check box.
    • Select the Receiving messages check box.
  • Specify account is for:
    • All.
    • Only for (select a user name from the list.)

Connection settings

Depend on email server settings.

Choose Account type:

  • IMAP.
  • POP.

Fill in other fields depending on the selected account type.

IMAP server settings

  • Enter the account Username in the format of User@Server.net and its Password.
  • Specify IMAP server, for example, imap.Server.net, Port, for example, 993.
    • Use radio button to set Encrypting:
      • Auto.
      • Always (SSL).
  • Specify SMTP server, for example, smtp.Server.net, Port, for example, 465.
    • Use radio button to set Encrypting:
      • Auto.
      • Always (SSL).
  • If necessary, select the When sending emails, authorization on the outgoing email server (SMTP) is required check box.

POP server settings

  • Specify the account Username in the format of User@Server.net and enter its Password.
  • Select the Keep email copies on server check box if you do not want the incoming messages that you just downloaded to be deleted from the mail server. The check box is cleared by default, in this case all emails will be deleted from the email server after receiving. After doing so, the check box becomes available:
    • In the Delete messages from server after field, specify how long you want the messages to be stored on the mail server. Example: 10 days.
  • Specify POP server, for example, pop.Server.net, specify Port, for example, 995.
    • Set up Encryption:
      • Auto.

      • Always (SSL).

  • Specify SMTP server, for example, smtp.Server.net, Port, for example, 465.

  • Use radio button to set Encrypting:

    • Auto.
    • Always (SSL).
  • If necessary, select Send email BCC to this address.
  • If necessary, select the Authorization is required check box and choose, using the radio button:
    • on outgoing email server (SMTP),
    • on incoming email server (POP).

Validating settings

  • Click Validate settings to validate email account parameters.

  • The application sends a standard message and checks connection to incoming mail server (if the account is used for receiving email.) You can also use the More - Validate settings command to the same effect.

Specifying external user properties.

To enter or edit external user details, you need administrator rights or external user list manager rights.

The Allow authorization right can be set by administrators only.

An external user list manager can edit the External users list (to a limited degree), configure external users prior to their authorization approval by administrator, use the External access command, edit the External users list and the External user groups lists, and edit access rights of external users.

Other users are only allowed to view their personal data, change their Password (if not restricted by administrator), edit their contact information, and select application interface language.

In a distributed infobase, authorization is configured for each infobase separately.

Recommendations on creating passwords

A strong password:

  • Is not human-readable.
  • Contains at least 8 characters.
  • Includes:
    • Uppercase Latin letters.
    • Lowercase Latin letters.
    • numbers
    • Special characters (such as underscore and parentheses).

Passwords that contain numbers only, human-readable words, or periodic character sequences, as well as passwords identical to usernames are not recommended.

  • Examples of strong passwords:
    “nj7{jhjibq*Gfhjkm
    F5”njnGhkmNj;t{HI
  • Examples of weak passwords:
    Smith
    qwerty
    12345678
    123123123

Creating external users (for users with sufficient rights)

After a user list manager creates an external user and specifies their username, a recommendation to approve their authorization settings is displayed to administrators in the external user profile. Once an administrator approves authorization for this user, the recommendation is hidden and the manager is no longer able to change the authorization settings in any way, with the exception of revoking the authorization approval. To view the list of external users that require approval, filter the external user list by the Requires approval field.

  • Each external user is linked to an application object (for instance, counterparty or employee) specified in the Object field. Click External access (recommended) to have the Object field filled in automatically.
  • If you are creating a external user using the Create command in the external user list, fill in the Object field manually. To perform this action, click Select, select a list, and then select an object from the list. The Object field value will be displayed in application lists as a full name of an external user.
  • The Invalid check box is cleared for all external users by default. If an external user no longer uses the application yet cannot be deleted due to being referenced by other objects, it is recommended that you select the Invalid check box for them. Invalid external users are not displayed in the external user list, and they cannot be selected.
  • On the Main tab, fill in the main external user details:
    • Username. A short name used to sign in to the application.
    • 1C:Enterprise authentication. This is the default option.
      • 1C:Enterprise authentication status is displayed under the check box.
    • To set a password, click Set Password and enter it. Never leave this field blank as it compromises security.
      • If the external user is not saved, you will be prompted to save it. Click Save.
      • Once the password is set, the authentication status changes to The password is set and the button name changes to Change password.
    • Require to set a password upon authorization. Select this check box to require the user to set a password when they sign in for the first time.
    • User cannot change password. Select this check box to prohibit the external user from changing their password.
    • OpenID authentication. Select this check box to allow the external user to sign in using an OpenID account. If you select this check box, you can clear the 1C:Enterprise authentication check box.
  • Use the Comment tab to write down additional information about external users.

Adding external users to groups

  • If the application supports user groups, you can add external users to one or several groups during the user creation procedure.
  • Click Groups on the navigation panel, and select the check boxes for one or several groups.
  • To clear all selections quickly, click Remove from all groups.

Access rights of external users

  • Access rights can also be set up by the external user list manager, to a limited degree (excluding administrative rights). Use one of the following options, provided that it is available in the application:
    • Click Access rights on the navigation panel to add an external user to one or more access groups. Select the groups from the list.
    • In the Allowed actions (roles) list, assign one or several roles to the external user, thus defining their personal access rights settings. The roles can be linked to job duties of users or to smaller activities. Select the check boxes for the roles you want to assign.

Allowing authorization (for administrators only)

  • To allow an external user to sign in to the application, select the Allow authorization check box. Then set the username, password, and other authentication parameters.
  • You can deny user access to the application manually by clearing the Allow authorization check box, or have the access denied automatically at the end of the specified period.
  • Open user sessions are closed automatically in up to 25 minutes after the user access is denied.

Authorization restrictions

Follow the link to edit authentication restriction. Select one of the following:

  • Apply general authorization settings. This is the default option.
  • No time limit.
  • Allow authorization till. Enter the date or click to select it from the calendar. To prevent unauthorized access to the application, you can set an Expiration date. Upon reaching this date, the external user is denied access to the application.
  • Deny authorization if no activity for (specify the number of days). If an external user does not sign in for more than the specified number of days, they are denied access to the application.
    • In this case the external user has to contact the administrator to restore access to the application.
  • Restrictions are listed in user profiles. Restrictions for all external users are also listed in the External user details report.

Managing external user settings

  • The application stores individual settings of each user for further use. To view, copy, or clear settings of external users, click Settings on the navigation panel. Clearing might be required when invalid settings prevent normal operation of the application.

Application warnings

  • If properties of an external user differ from the properties of the matching infobase user, the application displays a warning including a list of differences. Then it prompts you to save the data in order to resolve the differences.

Use this list to store external users (users working with the application "from outside"). Access rights of external users are usually strictly limited. External users do not actively participate in accounting, work independently, and do not intercommunicate. Examples of external users are: employees in an HR database, online store customers, and questionnaire responders.

You can open the list of external users from section Administration—Users and rights settings. Alternatively, you can open it from section Questionnaires (Access management).

External users are different from regular users, therefore, they are stored in separate application lists, for example, Counterparties, Individuals, and other lists.

Only administrators and external user list managers can edit the external user list.

Authorization of an external user requires administrator's approval.

External user list managers have the right to add external users and edit them (to a limited degree). They configure external users prior to their authorization approval by administrator. External user list managers can run the External access command, and also edit the External users and External user groups lists.

External users created by list managers are assigned the Requires approval status. Application administrators can filter users by this parameter, verify accuracy of the entered data, and allow authorization.

Once an administrator approves authorization for a user, list managers will be unable to change their authorization settings in any way, with the exception of revoking the authorization approval.

External users can view their personal data and edit personal parameters, such as password, application interface language, and contact information.

In a distributed infobase, authorization is configured for each infobase separately.

Creating external users

  • The External access button located on the command bars of the application lists is a primary tool for creating and editing external users. The command creates an external user linked to the selected object, or displays details of an existing external user.
  • You can also create an external user directly in the external user list. Click Create, then specify the Object linked to the external user.

Filtering external users

  • You can filter the list by external user type (for instance, partners).
  • In the Type field, click the button to open the selection window, and then select the check marks for the external user types you want included in the list.

Viewing external users that belong to subgroups

  • To view external users in the subgroups (all nesting levels), select the Show users that belong to subgroups check box at the bottom of the list. By default, the check box is selected.

Viewing invalid external users

  • Any external user can be marked as invalid. To view invalid external users, select the Show invalid users check box at the bottom of the list. The check box is cleared by default.

Viewing infobase users

  • You can check user data accuracy or modify incorrect user data. To synchronize the external user list with the infobase user list, click More actions—Infobase users command.

Grouping external users

  • If the application supports user groups, the list consists of two sections: the left one displays groups, and the right one displays the external user list. When you select a group, all external users belonging to this group are displayed in the right section.
  • Long lists of external users can be grouped for convenience (for instance, by user relation types, such as partners and contractors).
  • An external user can belong to multiple groups.
  • To add an external user to a group, you can use any of the following methods:
    • When creating an external user, select a group from the list and click Create.
    • To add multiple external users to a group quickly, open the group window and click Select. The users will be selected from the External users list containing three sections.
    • Select an external user from the list and use the More actions—Assign to groups command to include the user into one or more groups. You can also right-click a user to open a menu with this command.
    • Drag one or more external users to a group. To allow this feature, clear the Show users that belong to subgroups check box.

Creating external user groups

  • Click Create group and fill in the fields.
  • You can also right-click anywhere in the left section and then click Create.

Moving external users groups to other groups

  • Select a group in the left section of the list, right-click it, and then click Move to folder.
  • Select a destination group.

Selecting members of external groups

  • The list intended to select external users consists of three sections. Groups are displayed on the top left, external users belonging to the selected group are displayed on the bottom left, and selected users are displayed on the right.
  • For your convenience, available external user groups are listed in the top left section. Once you select a group, all members of this group will be listed in the bottom left section.
  • Select one or more users from the bottom left section of the list and click the arrow button to move them to the Selected users and groups list on the right. You can also double-click any external user to move them to the list on the right.
  • To remove users from the list on the right, select them and click the arrow button, or double-click each user.
  • Click the double arrow button << to remove all external users from the Selected users section.
  • The number of selected external users is displayed.

Viewing the External user details report

  • Use this report to analyze authorization settings (infobase user properties) and other external user properties. There are 3 report options available: a report on users, external users, or all application users.
  • Click More actions—External user details and generate the report.

Setting a password

This option is available only to administrators.

  • Click More actions—Set password and fill in the fields.

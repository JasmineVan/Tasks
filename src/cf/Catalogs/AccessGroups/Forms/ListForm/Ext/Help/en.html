The access group list is used to manage access groups and view the rights and restrictions settings.

To access this list, execute the Access groups command located in the Administration - User and rights settings section. For quick access to the list of access groups based on a specific profile, execute the Access groups command located on the navigation panel of the access group profile.

An access group defines a set of actions with the application data the members of this group are allowed to perform. Generally, access groups correspond to user job functions or duties. A user can be a member of multiple access groups at the same time, these groups collectively defining their personal access rights settings.

Access groups can be used to create and modify common access rights settings for a user group performing similar activities in the application, for example, "Headquarters accountants", or "New York branch sales managers".

You need the Administrator rights to manage the access. A user responsible for an access group can change the group's content.

The list displays:

  • Name - the access group list, sorted in alphabetic order.
  • Profile - for your convenience, access group profiles can be displayed in the list.
  • Person responsible. The user responsible for the access group.

Creating access groups

  • Select Create and fill in the required fields.

Creating access group folders

  • The list provides the ability to create folders (groups) to simplify the search for the required access group and to understand the access right restrictions in general, because there can be many access groups with similar settings of rights restrictions.
  • Click Create group and type in the folder Description.

Changing access group contents

  • Select the needed access group, click More - Change.

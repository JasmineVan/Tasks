Details of access group properties.

An access group defines a set of actions with the application data the members of this group are allowed to perform. Generally, access groups correspond to user job functions or duties. A user can be a member of multiple access groups at the same time, these groups collectively defining their personal access rights settings.

Access groups can be used to create and modify common access rights settings for a group of users performing similar activities in the application, for example, Accountants.

You need the administrator rights to manage the access. A user responsible for an access group can change the group's content on the Members tab.

An access group is always linked to an access group profile commonly incorporating multiple roles. Once a user is added to an access group, they are automatically assigned all roles listed in the access group profiles. For example, a predefined access group Administrators is linked to the Administrator profile incorporating the Full rights role. This role grants unlimited access to all data; it is assigned to all application administrators belonging to the Administrators access group.

Creating access groups

To create an access group, you need to select a profile (we recommend that you start from this point), specify group member list, and define any access restrictions.

  • Select Profile. It determines available rights (according to its role list) and available rights restrictions broken down by access kind: for example, restrictions related to Companies, Warehouses, etc. Any access kinds found in the profile will be added to the access group automatically.
  • Enter access group Name that should reflect its purpose and restrictions by access kind (for more information, see access group description).
    • We recommend that you choose a plural name containing the profile name (as displayed in the Profile field) and clearly describing the access right settings assigned to this access group. For example, you can create access groups named Sales managers in Forward Thinking Inc. and Sales managers in New Tech LLC on the basis of the Sales managers profile.
  • You can add access group to an access group folder.
    • Select a folder from the list in the Group (folder) field.
  • Allowed participants of the access group are defined in the profile in accordance with the selected assignment.
    • Depending on the profile settings, the access group may include different types of users: users and user groups. You can also add external users and external user groups to the access group (this may be useful for polls purposes, etc).
  • On the Participants tab, fill in the list of participants of the access group, indicate the person responsible for the access group.
  • Go to the Access restrictions tab and enter additional access rights settings.
  • You can also enter detailed information on the access group on the Commentary tab.

Group participants

  • Enumerate the list of users (and user groups) to which access rights settings should apply. Lists from which you can select group participants are listed in the Allowed participants field at the bottom of the tab.
    • To add multiple users, click Pick; to add a single user, click More - Add. Select the required access group members from list.
    • To delete one or several access group participants, select them and click Delete.
  • Fill in the Person responsible field by selecting a user from the Users list. The user assigned as a Person responsible for the group can change the list of group members.

Access restrictions

  • On the Access restrictions tab, you need to enter additional access rights settings. Access restrictions can only be created on profile level. On access group level, access restrictions can only be elaborated, if allowed by the profile settings. In certain cases, profile-based access restrictions are complete and require no elaboration. In these cases, the Access restrictions tab will be disabled.
  • Content of the access kind list on this tab depends on the profile specified in the Profile field. In addition, the profile also determines how the access kind setting works: it only allows viewing or it also includes the availability to edit data. For example, if the profile of access groups includes the role for reading the Demo: Goods receipt, users are allowed to view documents for the organizations specified. If the profile contains a role for adding and editing, then it becomes possible to edit such documents.
  • The Access kind field is used to precisely define the data area accessible to the access group members. Access kind is a some kind of rule that allows access to the application data. For example, using the Demo: Companies access kind, you can allow users to work only with documents on specific companies, or hide documents of some companies from them.
  • To set up the access restrictions, you need to specify a value list for each access kind.
  • To set the values, double-click the mouse (you can also execute the Edit command of the context menu by the right click of the mouse). Specify Access values for each Access kind. Two values are available:
    • All prohibited. In this case, access to all data is prohibited by default; you will need to explicitly specify any allowed values. We recommend that you use this method when you need to grant the users access to a specific limited data area.
    • All allowed. In this case, access to all data is allowed by default; you will need to explicitly specify any prohibited values. The second setting method is convenient in cases when you need to lock access to a previously known data area, and to all new values entered into the program, the allowing rules should be applied by default. For example, if using the Demo: Companies access kind the right is set that prohibits to work with the documents by the Forward Thinking company, these restriction will not be applied to the new company New Tech, added to the application later. Please keep in mind that "allow" settings have priority over "prohibit" settings. So, if in the example above, the user is allowed to enter documents on the Forward Thinking company in some other access group with the same profile, then in the end the user will get access to all documents on this company
  • You can specify a list of values or value groups to serve as exceptions for any Access kind, if necessary. For this, click Add in the Allowed (Prohibited) values list. Select exceptions from lists.
    • In some cases, to simplify the setting of access rights, you can specify not individual elements, but the entire groups of elements in the list of allowed or prohibited values. For example, for the Demo: Partner groups access kind you can restrict the access in terms of partner access groups, not by each partner separately. For this, you will need to create the list of partner access groups, and assign each partner to a group. These activities are not limited to administrators; any user with sufficient rights to edit the partner list can perform this.
  • For some access groups, the Users (or External users) access kind can be used for access rights setup. This applies only to the documents (and other application objects) containing fields that can be used to specify users allowed to access these documents (such as "Person responsible," "Author," and "Performer").
    • To grant group members access to all restricted data available to a specific user, you need to add this user to the allowed value list for the Users access kind.
    • For example, if access to the Receipt of goods and services documents depends on who is listed in the Person responsible field, then the user will be able to save documents for which the Person responsible is the same as the user, or added to the list of allowed for the Users type of access. The prohibiting setting acts in the same way.

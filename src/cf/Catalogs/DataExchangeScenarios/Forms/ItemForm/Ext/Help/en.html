This scenario is intended to perform automated data synchronization according to schedule. The scenario must be used to create a schedule, or to specify data sending and receiving procedure.

The scenario can be used to send data or to receive data separately, but we recommend that you use it only to send and receive data together. We also recommend that you perform full data synchronization in a single scenario.

Configuring synchronization scenarios

  • Enter scenario Description. You should specify all databases participating in synchronization, and program revision, in order to make the purpose of this scenario clear to all users.
  • Set a data synchronization scenario schedule.
    • Select the Auto scheduled check box. This will enable a link for selection (default value - Schedule not set). Click the link to set the schedule. Once you confirm the schedule, the link will change to display the schedule contents.
  • Select infobases participating in the data synchronization scenario (generally, a scenario includes data exchange - both sending and receiving data - with a single infobase). For each infobase, select connection method and current action (sending or receiving data). Click Add and fill in the relevant columns:
    • Infobase - Select data type. Then, select an infobase from the list.
    • Connection - select a connection method from the list (depends on the selected infobase properties);
    • Action to execute - use the button to Select whether to Receive or Send data;
  • Enter your Comment - a brief scenario description.

Executing scenarios

  • Click Run scenario. Data synchronization will be performed. All actions included in the scenario will be performed in order. After the data synchronization is complete, all failed data sending or receiving attempts will be marked with in the Infobase column.

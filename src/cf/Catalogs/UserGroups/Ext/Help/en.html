Use this list to manage user groups.

User groups can be convenient when a large number of users work with the application:

  • To group large numbers of users (this simplifies user selection).
  • To quickly configure access rights settings for all users in a group (for example, if you add all accountants to the Accountants group, you can set up access rights for specific lists and documents for the whole group at once, not each accountant separately).
  • To set up other group-level parameters, such as business process task performer roles.

You can create groups as a flat list (all groups are independent), or as a hierarchical list with parent and subordinate groups. In a hierarchical list, members of a subgroup also belong to parent groups.

The predefined group All users contains all application users.

If the explicit assignment of roles is available, you can assign the roles both to a single user and a user group.

Grouping users

You can create user groups containing users of a specific type. This approach facilitates the setup of access rights. For example, users can be divided into groups in accordance with corporate organizational structure, job titles, or project team membership:

  • Company
    • Administration
      • Department 1
      • Department 2
    • Administration managers
    • Engineering and technical services
      • Department 1
      • Department 2
        • Group 1
        • Group 2
      • Engineering and technical service directors
    • Manufacturing
      • Manufacturing shop 1
        • Shop area 1
        • Shop area 2
      • Manufacturing shop 2
        • Shop area 1
        • Shop area 2
        • Shop area 3
      • Manufacturing managers
    • Corporate executives
    • Joint inventory group

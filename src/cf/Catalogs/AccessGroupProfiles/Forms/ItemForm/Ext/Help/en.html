Description of access group profile properties.

A variety of predefined access group profiles is available in the application. However, in certain cases, predefined access group profiles prove to be insuffucient for the purpose of comprehensive setting of user access rights. In such cases, the administrator has the ability to add new access group profiles.

Generally, an access group profile contains several roles and access kind details that can be used by the administrator to set up user access rights via access groups. In other words, an access group profile is a template used by the administrator for access group management purposes.

One or several access groups can be linked to a single profile. For example, two access groups: Headquarters accountants and New York branch accountants, can be created on the basis on the Accountant profile which differ from each other by allowed organizations: Headquarters and New York branch.

Creating access group profiles

  • Enter Profile description.
  • If you need to add an access group profile to a folder, fill in the Group (folder) field. Access group profiles can be grouped by a variety of criteria. Select the required folder from the list.
  • In the Purpose field, click the link and use the checkboxes to select who the access group profile is designed for, for example, Users, and then click Select. After this the link will receive the selected value.
  • Select Allowed actions (roles) on the corresponding tab.
  • Specify Access restrictions on the corresponding tab.
  • You can write additional information about the access group profile on the Сomment tab.
    • The predefined access profiles have the Details tab for the same purpose that contains details of this profile and its purpose.

Allowed actions (roles) 

  • When creating a new access group profile on the Allowed actions (roles), you first need to assign a list of roles allowed for this profile, according to the demands of the target users.
  • A role defines a set of actions with the applied objects the user is allowed to perform. Roles can correspond both to different job responsibilities (or types of activities) of application users, and to smaller functions. For example, the Accountant role grants access to a set of functions needed to perform duties of accountant, while the Read files role allows viewing file contents. Roles are often used to allow viewing or editing specific data stored in the application.
  • Select one or several roles, using the check boxes.
    • To select all roles, click .
    • To cancel the selection of all roles, click .
  • Click Selected roles only to show selected roles only.
  • By clicking the More - By subsystems command you can view the list of roles in one of the following ways:
    • sorted alphabetically;
    • grouped by subsystem.

Access restrictions

  • The Access restrictions tab is used to assign a list of access kinds to the profile.
  • Access kind is a some kind of rule that allows access to the application data. Access kinds are used to precisely define the data area allowed to the access group members.
  • In an elementary case, no access kinds are assigned to a profile and data access rights are solely governed by profile roles. For example, if the profile includes the role for reading documents Demo: Goods receipt, then all group participants are allowed to view all documents. However, once you add the Demo: Companies access kind to this profile, you will have the option to allow the users to view documents on a specific company only, or to block the users from viewing documents on a specific company.
  • To reiterate, the roles assigned to a profile define which data or functionality is accessible and which is not; while the access kinds assigned to a profile allow to define more precise conditions for data accessibility.
  • We recommend that you assign access kinds to a profile whenever you need to specify complicated access rights for the group members.
  • Specify the Access kinds.
    • Click Add and select an access kind from list.
    • You can Delete unnecessary records using the corresponding command in the More menu, or right click it from the context menu.
  • For each access kind assigned to a profile, you can select any of the four available Access values:
    • All prohibited, exceptions defined in access groups - select this access value to have access to all data prohibited by default. To allow access to an object, an exception must be created in an access group.
    • All allowed, exceptions defined in access groups - select this access value to have access to all data allowed by default. To prohibit access to an object, an exception must be created in an access group.
    • All prohibited, exceptions defined in profiles - select this access value to have all data access prohibited by default. Data access exceptions can be created in profiles only; access rights cannot be modified on access group level. These access kinds will be hidden from the access groups.
      • When this access value is selected, exception creation functionality will be enabled.
      • Select Add, and select values from lists.
      • You can Delete extra values using the matching command in the More actions menu, or right click it from the shortcut menu.
    • All allowed, exceptions defined in profiles - select this access value to have all data access allowed by default. Data access prohibition exceptions can be created in profiles only; access rights cannot be modified on access group level.

View unused access kinds

  • To view those access kinds, that are not used at the moment (for example, this feature is disabled in the application), click More - Show unused access kinds.

Viewing access groups

  • For quick access to the list of access groups based on a selected profile, execute the Access groups command located on the navigation panel.

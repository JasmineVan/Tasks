This form is used for access group profile management.

To facilitate management of multiple access group profiles with similar purposes and avoid confusion, you can create folders to simplify the search for the required profile and understanding the principles of their generation.

To access the list, execute the Access group profiles command in the Administration - User and rights settings section.

List filters

  • Using the Show field you can display:
    • All profiles (by default),
    • Supplied - profiles, that are included in the application delivery,
    • Not supplied - profiles, that are not included in the application delivery,
    • Obsolete.
  • Using the Purpose field, you can select the access group profiles by the kinds of users for whom they are intended:
    • Then click Select.
    • Use the checkboxes to specify who the access group profiles are for, for example, Users.

Main and additional access group profiles

  • There are main and additional access group profiles:
    • Primary profiles define a set of access rights sufficient to perform specific activities in the application.
    • Using the additional profiles, users can be given any auxiliary rights in addition to the main profile.
      • For example, in addition to the Accountant and Warehouse supervisor main profiles, there are Counterparty communication and Unposted document printing profiles.

Recommendations on creating primary access group profiles

  • A primary profile should include a set of roles that grants all access rights required by users during their routine duties and operations, but does not provide them with excessive or unadvisable access to the application data or functionality.
  • Moreover, a primary profile should include additional roles not directly related to the user duties but yet necessary to perform these duties.
  • For example, the following roles must be included in the profile intended for document entry operators, besides the roles allowing to create and edit these documents:
    • A role granting the right to sign in using the thin client, the thick client, or the web client.
    • the Basic rights and TM basic rights predefined roles,
    • roles allowing to view catalogs containing items selectable in the document fields;
    • roles allowing to view document processing reports, etc.
  • For example, the Auditor profile should include the Read individuals primary role to allow viewing personal data on individuals, and a set of additional roles to support basic work functionality (such as Basic rights, Start thin client, etc).

Creating access group profiles

  • Select Create and fill in the required fields.

Creating access group profile folders

  • Click Create group and fill in the fields.

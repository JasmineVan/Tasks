Use this list to map infobase users added in Designer to the members of Users and External users lists.

To open the list, use the More actions—Infobase users command in the user list or the external user list.

When you save a user or an external user, a matching infobase user is created or modified automatically. However, in certain cases, discrepancies may occur between the infobase user list and the application user list. This list can be useful when you need to:

  • Find "incorrect" (unmapped) infobase users:
    • The application compares the infobase user list with the application user list.
    • If an application user was added, modified, or deleted in Designer, the mapped infobase user becomes inactive (marked gray).
    • Infobase users that cannot be mapped to application users are marked red.
  • Find mapping errors.
    • If an infobase user was created in Designer or programmatically, map it to an application user or Delete it.
    • If an infobase user was deleted in Designer or programmatically but is still referenced by an application user, open the application user and save it in order to clear the reference (or map it to another infobase user).
    • If a username of an infobase user was modified in Designer or programmatically, open the user profile and save it to restore the full name of the infobase user.
      • In this case, the user profile shows a warning that differences must be eliminated. There are two buttons below the warning:
        • Fill from infobase user. The Full user name field will be filled with the changed name from the Infobase users list.
        • Save. The full name from the user profile will be written to the Infobase users list.
  • View the list of all infobase users (this list is also available in Designer).

The list displays:

  • Full name. A username of an infobase user (might differ from the username in the Users or External users list in case discrepancies occur).
  • Username. Might differ from the username in the Users or External users list in case discrepancies occur.
  • 1C:Enterprise authentication. The value is "Yes" if 1C:Enterprise authentication is enabled for the user.
  • OpenID authentication. The value is "Yes" if OpenID authentication is enabled for the user.
  • OS authentication. The value is "Yes" if OpenID authentication is enabled for the user.

You can add the following columns using the More actions—Change form command:

  • OS user. The operating system username for signing in to the application if OS authentication is enabled.
  • Mapped to external user. Shows whether it is an external user.
  • Added in Designer. Shows whether the user is added in Designer.
  • Modified in Designer. "Yes" if the user is modified in Designer. Otherwise, False.
  • Deleted in Designer. "Yes" if the user is deleted in Designer. Otherwise, False.

Viewing unmapped records or the entire list

  • To view unmapped records that require manual correction, select the Show only users added, modified, or deleted in Designer check box. To view the entire list of infobase users, clear this check box.

Mapping users

To map infobase users to application users, use one of the following buttons:

  • Map. Map an infobase user to an existing application user.
    • Click User or External user and click OK.
    • To select a user, double-click it, or click it and then click Select.
    • In the User or External user window, make edits and click Save and close. After that, user data mapping in two lists will be restored.
  • Map to new user. Map an infobase user to a new application user.
    • Click User or External user and click OK.
    • Create a user or an external user. After saving data user data mapping in two lists will be restored.
  • Clear mapping. Clear automatic mapping of infobase users to application users for further manual mapping. This is only required in rare cases when mapping was performed incorrectly (for example, during the infobase update). It is recommended that you do not clear the mapping in other cases.

Viewing mapped users

  • Select an infobase user. To open an application user or an external application user mapped to the infobase user, right-click the infobase user or click More actions, and then click Go to user.

Use this list to store users working with the application.

You can open the list of users from Administration—Users and rights settings section or from Access management—Users section.

Only administrators and external user list managers can edit the user list.

Authorization of an external user requires administrator's approval.

User list managers have the right to add external users and edit them (to a limited degree). They configure users prior to their authorization approval by administrator. User list managers can edit the Users and User groups lists.

Users can view their personal data and edit personal parameters, such as password, application interface language, and contact information.

Users created by list managers are assigned the Requires approval status. Application administrators can filter users by this parameter, verify accuracy of the entered data, and allow authorization.

Once an administrator approves authorization for a user, list managers will be unable to change their authorization settings in any way, with the exception of revoking the authorization approval.

In a distributed infobase, authorization is configured for each infobase separately.

Long lists of users can be grouped for convenience. For instance, companies can use user groups that copy their organizational structure.

You can also right-click anywhere in the list to open a menu with user list management commands.

Creating users

  • Click Create and fill in the fields.

Viewing invalid users

  • Any user can be marked as "invalid." To view invalid external users, select the Show invalid users check box at the bottom of the list.

Viewing users that belong to subgroups

  • To view users in the subgroups (all nesting levels), select the Show users that belong to subgroups check box at the bottom of the list. By default, the check box is selected.

Setting a password

  • Click More actions—Set password and enter a password.

Viewing the list of infobase users

  • You can check user data accuracy or modify incorrect user data. To check records or synchronize the user list with the infobase user list, click More actions—Infobase users command.

Grouping users

  • If the application supports user groups, the list consists of two sections: the left one displays groups, and the right one displays the user list. The All users group is the default current group. Use it to view all application users.
  • A user can belong to multiple groups. To add a user to a group, use any of the following methods:
    • When creating a user, select a group from the list and click Create.
    • To add multiple users to a group, open the group window and click Select. The users will be selected from the Users list containing three sections.
    • Select a user from the list and use the More actions—Assign to groups command to include the user into one or more groups. This command is also available in the context menu.
    • Drag one or more users to a group. Click Yes to confirm the selection. To allow this feature, clear the Show users that belong to subgroups check box.

Creating user groups

  • To create a group, click Create group.
  • You can also right-click anywhere in the left section and then click Create.

Moving user groups to other groups

  • Select a group in the left section of the list, right-click it, and then click Move to folder.
  • Select a destination group.

Selecting user group members

  • The list intended to select users consists of three sections. Groups are displayed on the top left, users belonging to the selected group are displayed on the bottom left, and selected users are displayed on the right.
  • For your convenience, available user groups are listed in the top left section. Once you select a group (or the All users group), all members of this group will be listed in the bottom left section.
  • Select one or more users from the bottom left section of the list and click the arrow button to move them to the Selected users list on the right. You can also double-click any user to move them to the list on the right.
  • To remove users from the list on the right, select them and click the arrow button, or double-click each user.
  • Click the double arrow button << to remove all users from the Selected users section.
  • The number of selected users is displayed.
  • Once you have finished selecting group members, click the Apply button.

Viewing the User details report

  • Use this report to analyze authorization settings (infobase user properties) and other user properties. There are 3 report options available: a report on users, external users, or all application users.
  • Click More actions—User details and generate the report.

This list is used for file storage volume management.

It is only utilized when the In volumes on hard disk file storage mode is active. To access this list, select the Store files in volumes on hard disk check box located in the Administration - File operation settings section. This will enable the File storage volumes command located in the Administration - File operation settings section. Execute this command to open the list. If you change the file storage mode, the files will not be moved from the application to the volumes or vice versa.

The list includes the following columns:

  • Name. Name of the file storage volume.
  • Order. An integer. Storage volumes are filled in order (ascending) until each volume is full.
  • Full path for Microsoft Windows. Path to the volume folder on a Microsoft Windows server in the UNC format: \\Computer name\Folder name\File name
  • Comment. Detailed information on the volume.
  • Limit, MB. You can specify the maximum size of all the files in the volume (volume quota). If the quota equals zero, it will not be checked upon placing files in the volume. If the Max size does not equal zero, when total size of all files stored in a volume exceeds the quota set for this volume, the next volume in fill order will be used for file storage. If none of the available volumes have sufficient free capacity for file storage, this event will be added to the event log and an error message will be displayed.
  • Full path for Linux. Path to the volume on Linux server.

Creating file storage volumes

  • Select Create and fill in the required fields.
  • You must specify one or several folders located on hard disks of server running the application or other servers in the network, to be used for file storage.

Rules for storing files in volumes

  • If you select or clear the Store files in volumes on the hard disk check box, you will no be able to cancel your changes. If you disable volume file storage, new files will be added to the infobase while the old ones will remain in volumes. A volume folder will be used until the infobase stores at least one file. Files that were previously saved to volumes open from these volumes.
  • If you save a file in a volume, and then delete the volume by external means, the file will not be found even if the Store files in volumes on the hard disk check box is already disabled. Selecting or clearing the check box will not bring the missing files back.
  • If the application cannot find the files stored in volumes, you should look for error messages in the event log, see the path to the volume folder, check the location of this folder, make sure it contains the file you need, and restore the folder if necessary. To be able to restore folders, make sure you perform regular back-ups of volume folders by external means, for example, by operating system means.

Managing file storage volumes

  • When creating a new storage volume, access to folder specified in the Full path field is validated automatically. If the path is invalid, a warning message will be displayed.
  • Storage volumes are filled in order (ascending) until each volume is full. You can use this to temporarily disable a volume by increasing its filling order.
  • Path to an existing volume cannot be modified.
  • A volume cannot be deleted while it is being referenced by any file versions.
  • Volume access errors are recorded to the event log and can be viewed by the application administrator.
  • Files in each volume are stored in subfolders organized by date. A new subfolder is created for every day. All stored files retain their original names and extensions, with version number added in the middle. Example: original file name Orders.txt, version number 1, resulting file name Orders.1.txt.
  • Toggling file storage mode does not affect the currently stored files; it only defines how the new files will be stored. For example, if you enable the In volumes on hard disk file storage mode, this does not move the currently stored files to volumes; only new files will be stored in volumes. To move the currently stored files from the infobase to volumes, use the Transfer files to volumes.
  • If data synchronization is enabled between two applications and the source application stores files in volumes on hard disk, the initial image creation procedure must include creation of a compressed file containing the file storage volumes.
  • Accordingly, the procedure of initial image import to the target application must include specifying location of the compressed file with the file storage volumes. To extract the volume-based files from the compressed file, use the Store initial image files command.
Note
To start using volumes located on network disks, you need to run the 1C:Enterprise 8 server on behalf of a domain user (such as domain1\user1) with access rights to volume location specified as UNC path. This user must be a member of the Users group on the local computer, have Run as service and Run as batch job rights, and have access rights to folders where system files of 1C:Enterprise 8 server (event log, temporary files, etc) are stored.

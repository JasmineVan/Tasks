Description of file storage volume properties.

When creating a new storage volume, access to folder specified in the Full path field is validated automatically. If the path is invalid, a warning message will be displayed. You cannot change path to an already created volume.

These fields are mandatory for each volume:

  • Name. Full name of file storage volume.
  • Code. Filled in automatically.
  • Full path for 1C:Enterprise server under Microsoft Windows. Specify a path to the volume folder on Microsoft Windows server, in UNC format (\\Computer name\Folder name\File name\).

or

    • Full path for 1C:Enterprise server under Linux. Specify a path to the folder on Linux server.
  • Order. Enter an integer. The default value is 1. Storage volumes are filled in order (ascending) until each volume is full. For example, you can use this to temporarily disable a volume by increasing its filling order.
  • Limit, MB. You can specify the maximum size of all the files in the volume (volume quota). When total size of all files stored in a volume exceeds the quota set for this volume, the next volume in fill order will be used for file storage. If none of the available volumes have sufficient free capacity for file storage, this event will be added to the event log and an error message will be displayed. If the quota equals zero, it will not be checked upon placing files in the volume.
  • Current size, Mb. Total size of files currently stored in the volume. Updated automatically as the volume fills up.
  • Use the Comment tab to write down additional information about the volume.

Specifics of filling in the Full path under Linux field

  • For 1C:Enterprise servers under Linux, you can use only the local path to the file storage volume. Example: /home/path. Tu use UNC path, make a reference to it. To do this, follow these steps as administrator:
    • mkdir folder, where folder is your folder's name. Example: enter the /home folder and execute mkdir shared. This will create the shared folder (full path: /home/shared).
    • Execute command mount -t cifs //server/share folder -o //domain/user, password=pwd, where
      - where //server/share is a UNC path to another computer; folder is a path to a directory on the Linux computer, for example, \home\shared, //domain/user is the domain/user name (if necessary), and pwd is the user password (if necessary).
    • If username and password are not required, simply execute mount -t cifs //server/share folder.
  • Now you can specify folder (example: /home/shared) as the path to the volume.

Viewing volume file lists

  • Files in each volume are stored in subfolders. For each file version creation date, a new subfolder is created. All stored files retain their original names and extensions, with version number added in the middle. Example: original file name Orders2.txt, version number 2, resulting file name Orders2.2.txt.
  • To view the list of files in volume, execute the Files in volume command located in the volume navigation panel.

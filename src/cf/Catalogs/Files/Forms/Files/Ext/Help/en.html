The application allows you to store files in the infobase without linking them to certain objects. The list is designed to manage such files.

Working with the list is similar to those in the Windows Explorer.

On the left side of the list, there is a hierarchical folder structure. The list contains predefined folders that cannot be deleted, for example, File templates folder.

On the right side of the list, you can see a list of files stored in the selected folder. You can Add new files and folders to the list, Copy, Edit or Mark them for deletion using commands of the context menu or More actions menu.

To work with file attributes (author, name, description, etc.), use the file profile (More menu - Open card). Working with the file itself (View or Edit file) is performed in the application installed on your computer.

When editing a file, its previous version is automatically saved in the infobase. This setting is set as default. We do not recommend that you change this setting because if a file was damaged during editing, you can go back to one of its previous versions.

Files that you lock for editing are highlighted in green. Gray color is used for files locked for editing by other application users.

The list includes the following columns:

  • . Shows whether the file has a digital signature, or it is encrypted.
  • Name. Full name of a file. An icon next to the name specifies file type. In the same column, you can see a full file description.
  • Author. Name of the user who added the file.
  • Date changed. Date and time a file version is added to the infobase.

By clicking More - Change form, you can add some fields, for example:

  • Edited. User who added a file version.
  • Current version. Code of the current version, etc.

Adding files

To add a new file to the application:

  • Select a folder.
  • Click Add (press Insert).
  • Select an option to create the file:
    • From disk
      • To attach a file to the object or put a file existing on your computer to the folder, specify a path to the file on the disk, and then click Open.
      • The application saves the file and automatically creates and opens a File profile.
    • From template
      • Choose a file template in the selection list, and then click Select.
      • The application automatically creates and opens a File profile.
    • From scanner
      • You can create a file From scanner if a scanner is connected to the computer (see scanner documentation) and you have a scanner component installed on your computer. You should also configure scan settings.
      • If all conditions are met and settings are set, you can get a file from scanner.
      • To do this, follow the Scanning wizard steps.
      • After the document is scanned, click Accept. A file is assigned a code, and a file profile is created. You can see this profile on your screen. In the Name field, the code is inserted automatically. Edit the necessary fields, click Write and close to save the file profile.

File templates

  • There is a predefined File templates folder in the application. This folder contains files used as a template to create new files. For example, you can put a prepared file with contract text to this folder, and then use it to create contracts for certain counterparties.
  • To add a new template to the folder, open the File templates folder, click Add, and then select From disk by using radio buttons. Choose a file on your computer.
  • The application automatically creates a file profile, and then displays it on the screen. Add a template description. The Folder field is filled in with File templates by default.

Placing files in folders

  • In the list, you can Create required number of folders and store files there that are not linked to particular application objects.
    • You can also create a folder by choosing a command in the More menu, or right-clicking the context menu.
    • Fill in the required fields.
  • You can drag and drop files to different folders of the list. If the objects are selected, you can click More - Move to folder.
  • Dragging folders, you can subordinate them to each other.

Access rights to folders

  • Folders can be assigned access rights of different level.
    • Access rights to file folders are assigned by application administrator or user having permissions to manage folder rights.
    • To assign access rights to a folder, use a required command on the navigation panel of the folder.

Importing files

  • To import files from a hard or network drive to a folder of the files list, select this command in the More menu, or right-click Import files in the context menu.
  • In the Windows Explorer, specify a file path, and then click Open.
  • Specify information required for import.

Importing folders

  • To import a folder from a hard or network drive to a folder of the Files list, select this command in the More menu, or right-click Import folder in the context menu.
  • In the Windows Explorer, specify a path to the folder, and then click Select folder.
  • Specify information required for import.

Importing files and folders by using Drag-and-Drop method

  • To import one or several files (folders), you can use the Drag-and-Drop method. Drag a file (folder) or several files (folders) from the Windows Explorer to a folder of the application files list.
  • One file is written to the files list at once, and a file profile is created automatically.
  • When dragging a folder, or several files or folders, fill in required information.

Exporting folders

  • To copy a folder with files to a hard drive, click Save folder in the folder context menu. Only active file versions are copied.

Managing file list view

  • To change the list kind and view all files located in different folders in one list, click More actions - Settings - Use hierarchy.

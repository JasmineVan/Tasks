External user group

Use this window to view and edit external user group properties.

You can group external users by various parameters. An external user group can include users of the same type, such as counterparties or individuals. You can also create groups including external users sharing a common property, such as survey respondents. This approach facilitates access rights setup.

Creating external user groups

When editing lists of external users in groups, keep in mind that such users also belong to parent groups.
  • Enter group Description.
  • On the Group members tab, set the mode of selecting external users. Groups can include:
    • All external users of the specified types (for instance, counterparties). The Select button becomes unavailable in this case as all external users from the specified lists are included into such groups automatically.
    • Selected external users of the specified types (for instance, partners). In this case users have to be selected, which is set by default.
      • By clicking the Type link, set the types of external users to be included into the group. In the selection window, select the check boxes to specify the types of external users.
      • Click Select to select multiple external users and include them into the group.
      • Use the More actions—Add command to include a single external user into the group.
  • To make the group subordinate to an existing group, in the Parent group field, select the parent group.
  • You can add a detailed description of the group on the Comment tab.

Specifying access rights of external users groups

Depending on the application design, the following ways to set access rights for external user groups can be available:

  • On the group navigation panel, click Access rights. Click Add to group and select an access group.
  • If the explicit assignment of roles is available, you can assign the roles both to a single external user and an external user group.

Viewing invalid external users

  • Any external user can be marked as invalid. Invalid external users are hidden from all lists by default. To view such users, click More actions—Show invalid external users.

The catalog is designed to configure synchronization settings with cloud services for all users and all application objects.

You can open it in the Administration - Configure file operations section by clicking Synchronization settings.

Warning! To synchronize files, you should sign up at a cloud service and set settings recommended by this service on your computer. For more information, see the instruction on the selected cloud service.

You can use features of cloud services when working with files stored in the application. Cloud services can be used to store and process files attached to different application objects, and those stored in the Files list.

The application allows you to view your files in your explorer folder, or on your device, and edit and synchronize them there.

List of cloud services

The list is limited and cannot be edited. The following cloud services are supported:

  • Yandex.Disk
  • 4shared.com
  • Box.com
  • Dropbox (via the paid service dropdav)

How file synchronization works

  • You can synchronize only one folder in one exchange plane.
  • If you have file synchronization configured with the cloud service, editing and creating new files in the application is disabled to prevent conflicts. You can continue editing files in the application if you disable synchronization with the cloud.
  • In the cloud service, files are stored in the folder according to the following template: <Share path to account>\Application object presentation\Data item presentation, for example, \YandexDisk\SL 2.3.2\Demo sales order (Document) \Demo Sales Order 00-0000001 dated 12/17/2010 17 46 36.
  • When creating new files or editing existing ones, files are automatically captured by the file Author specified in the account.
  • After synchronization is complete, files not precessed during synchronization are released.
  • After synchronization, all unused accounts are checked. If they captured files, files are released. For example, if you want to disable synchronization, clear all settings with the Synchronize check box. The next time you run the scheduled job, all files are released.
  • The RLS mode (access rights limitation on records level) is not supported.

Configuring file synchronization

  • There are the following columns in the list:
    • File owner. All application objects that are file owners. With check boxes, you can select files of file owners to synchronize.
    • Account. Cloud service account. You can specify different accounts for different file owners.
    • Rule. The link shows files that are synchronized. The default value is set to All files. By clicking the link, you can navigate to file synchronization settings of a certain owner.
  • Click Configure file synchronization, or choose a context menu command to add a new file synchronization for application lists. You can configure several synchronizations for such objects.

File synchronization

  • You can synchronize files with the cloud service manually. To do so, click Synchronize in the list.

Configuring automatic file synchronization

  • Select the Automatically synchronize files check box.
  • Click the Set schedule link, that becomes available. Set the schedule.
  • After you confirm the schedule, it is displayed under the check box.
  • Synchronization by using a scheduled job is performed each 600 seconds by default.

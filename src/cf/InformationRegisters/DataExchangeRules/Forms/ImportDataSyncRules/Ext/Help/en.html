It is designed to control and replace the set of rules for data synchronization.

To open it, click the Import rule set button from data synchronization setting.

The default data synchronization setting is applied in accordance with Standard exchange rules. The exchange rules include object conversion rules and object recording rules.

You can access the object conversion rules from the data synchronization settings, via command More actions - Import object conversion rules. Data conversion rules are used during data synchronization between two applications.

You can access object registration rules from the data synchronization settings, via command More actions - Import object registration rules.

Rule source

  • For standard data synchronization between standard applications, the standard rules from configuration are used.However, in some cases, it is necessary to use the From a file on a computer rules (for example, if synchronization is conducted between databases with non-standard applications).
  • The Rule source radio button is used to select a source for object registration rules:
    • From configuration (selected by default).
    • From file on computer.
  • Click Import and select path to a file. Ensure that the rules you are about to import are created in the current application version. Using rules created in previous versions can cause errors.
  • The list of currently active rules is displayed below the radio button.

Information about rule set

The following rules information is displayed for additional verification:

  • Reminder of errors that could occur due to using rules from external files;
  • Object registration rules of this infobase. Here a version of rules and their creation date are specified.
  • Object conversion rules of this infobase. Here a version of rules and their creation date are specified.
  • Correspondent conversion rules. Here a version of rules and their creation date are specified.

Debug mode

  • If a radio button is set to the From file on computer position, debug mode becomes available.
  • Click the Debug mode setting reference.
  • Select the Enable debug mode check box. After that, the following settings become available:
    • Attach external data processor for export debug. Specifies a file of external data processor containing an export handler debug module created in Data conversion application. In client/server mode, server path is specified.
    • Attach external data processor for import debug Specifies a file of external data processor containing an import handler debug module created in Data conversion application. In client/server mode, server path is specified.
    • Data synchronization logging mode. If this option is enabled, all events and errors that occur during data exchange will be recorded to a log file you select. If you do not select a log file, all messages will be recorded to Event log.
    • Do not interrupt data exchange on errors. If this option is enabled, data exchange procedure will continue uninterrupted upon any errors. Data exchange will complete, and any debug messages will be saved to the exchange log file or event log.

Importing rules

  • Switch the radio button to From file on computer.
  • Click Load.
  • Specify path to a file with rules.

Exporting rules

  • You can export the recording rules to a separate file.
  • To do that, click More actions - Save rules to file. Specify file path on a local computer or network disk. File name will be generated automatically. Click Save.

Using external rule files instead of standard object conversion rules

  • Switch the radio button to From file on computer and click Import to select a rule file located on a local computer or network location.
  • Please note that you will import the correct rules that are appropriate for the used versions of the source application and destination application. For this, information about rule set is displayed where the date of rules creation is indicated.
  • Click Save and close. The imported rules will apply immediately.

Note: the object conversiond and registration rules are used for all data synchronization of the same type (that are based on one exchange plan; for example, two data synchronizations: "Synchronization with the branch in Krasnodar" and "Synchronization with the representative office in Kiev" are based on the Full exchange plan). Therefore, to apply new rules to multiple data synchronization procedures of the same type, you only need to import new rules for any one of them; the new rules will automatically apply to all other data synchronization procedures of the same type.

Use this form to view and replace data recording rules. Data recording rules are required for all data synchronization procedures.

The default data synchronization setting is applied in accordance with Standard exchange rules. The Exchange rules include object conversion rules and object registration rules.

You can access object registration rules from the data synchronization settings, via command More - Import object registration rules.

Rule source

  • The Rule source radio button is used to select a source for object registration rules:
    • Use standard rules (selected by default).
    • Import from file on computer (for debugging purposes only).
  • The list of currently active rules is displayed below the radio button.

Rules information

  • The following rules information is displayed for additional verification: full description of the rules, the application version, the rules creation date.

Importing rules

  • Set the radio button to Import from file on computer.
  • Click Load.
  • Specify path to the rules file if necessary.

Exporting rules

  • You can export the recording rules to a separate file.
  • To do that, click Save registration rules to file,
  • Select export method:
    • Normal file.
    • Archivezip.
  • Specify file path on a local computer or network disk. File name will be generated automatically. Click Save.

Using external rule files instead of standard object recording rules

  • Set the Rule source radio button to Import from file on computer and click Import to select a rule file located on a local computer or network location.
  • Make sure that the imported files are correct and suitable for the source and destination configuration in use. Take note of rules creation date and other useful information displayed in the Rules information field.
  • Click Save and close. The imported rules will apply immediately.

Note: the object registration rules are used for all data synchronization of the same type (that are based on one exchange plan; for example, two exchanges: "Synchronization with the branch in Krasnodar" and "Synchronization with the representative office in Kiev" are based on the Full synchronization exchange plan). Therefore, to apply new rules to multiple data synchronization procedures of the same type, you only need to import new rules for any one of them; the new rules will automatically apply to all other data synchronization procedures of the same type.

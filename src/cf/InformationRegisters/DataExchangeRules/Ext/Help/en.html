The default data synchronization setting is applied in accordance with Standard exchange rules. Synchronization based on outdated rules is likely to result in critical errors and corrupted data. To avoid this, it is imperative to review the standard exchange rules in use whenever you set up a new data synchronization, or when an application participating in data synchronization is updated.

Standard exchange rules include object conversion rules and object recording rules. Data conversion rules are used during data synchronization between two applications. Recording rules are available for all kinds of data synchronization. A set of current exchange rules is published on the technical support website http://users.v8.1c.ru. Corrective updates to the exchange rules are also available on this website.

Checking exchange rules validity

  • When checking validity of the exchange rules, pay attention to the following parameters:
    • Release versions of source application and target application to which these rules apply;
    • rules creation date.

  • Validity check is performed as follows:

    • determine values of controlled parameters for the source application;
    • determine values of controlled parameters for the target application;
    • determine values of controlled parameters included in the source application distribution kit;
    • determine values of controlled parameters included in the target application distribution kit.
  • Review all rules and select the rules with the latest creation date. When reviewing the rules, pay attention to the configuration release versions. Configuration release versions specified in the rules must match release versions of the configurations participating in data synchronization.

Checking rules loaded to applications

  • Create a new data synchronization, but do not run it.
  • Open settings for this data synchronization.
  • In the More menu of the data synchronization setting, execute the Import object conversion rules command.
  • Validate information on the data conversion rules.
  • Then, execute the More - Import object registration rules command.
  • Validate information on the object change record rules.
  • If you want to review the controlled parameters loaded to the application for an existing data synchronization, the procedure will be similar. For example, to validate object registration rules for an existing data synchronization setting in a subordinate node, use the Import object registration rules command.
  • To validate rules for data synchronization between two applications, you need to perform this procedure first for the source application, and then for the target application.

Checking rules included in application distribution kit

  • Using the radio button, select From file on computer in the Data conversion rules (Data registration rules) window . Execute the Import command.
  • Select the conversion rules file located in the application distribution kit.
  • The file path may vary depending on a distribution kit. The Data exchanges folder is a common location for this file.
  • Validate the rules information.
  • You need to perform this procedure first for the source application, and then for the target application.

Checking rules from external files

  • Review the rules information and click Save conversion (registration) rules to file to save a copy of the current rules to a file.
  • Set the radio button to Use standard rules.
  • Review the available rules information.

Making decisions based on check results

  • As a result of rules validity check, the latest version is identified and a decision is made:
  • Continue using current rules.
    • No additional actions required.
  • Start using the latest rules available in one of the applications.
    • The rule import procedure is described in the file commonly named Exchange rules attachment instruction.txt, located in the Data exchanges folder.
  • Start using standard conversion rules and object change record rules instead of the external rules.
    • For this, execute the More - Import object conversion rules (Object registration rules) command. Then, set the radio button to Use standard rules.

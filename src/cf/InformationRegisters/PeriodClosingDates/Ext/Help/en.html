It is designed for setting the date of period-end closing or data import restriction.

Data import restrictions might not be used. In this case, only window for setting period-end closing dates is available.

You can set a period-end closing date and apply various related settings in Administration - User and rights settings  - Period-end closing dates.

You can set a data import restriction date by clicking the relevant command in the Data synchronization section.

Date of period-end closing (data import restriction)

  • Sometimes you might want to set period-end closing in the application until a specific date. For example, after submitting quarterly accounting reporting to tax inspection, period-end closing is usually set for the data that was used to generate this reporting, i.e. data until the last day of the accounting period.
  • Import restriction, unlike period-end closing, can be set for the data to be edited only in one infobase node and according to the rules of data synchronization requirements, not the standard company business processes. For example, a decision to edit data of some companies only in the head office (in the central node) to technically assign responsibility for this data to a specific head office employee is a basis for restriction of data import from other nodes.

Examples of period-end closing (import restriction)

  • If period-end closing date is set, whenever you try to change the application data before period-end closing date, a message appears showing that the data cannot be changed.
    • If there are objects marked for deletion in the closed period, you will not be able to delete them.
    • If a period-end closing date is not set, restrictions are not valid.
  • If a data import restriction date is set, whenever you try to get data to the application until the import restriction date, all the restricted data will be skipped, their information will be written to the event log, and permitted data will be received.

Setting period-end closing date (data import restriction date)

  • In the application, you can set or remove a period-end closing date (data import restriction date). To do so, select one of the following values:
    • Not set - all period-end closing dates (data import restriction dates) will be removed.
    • For all users (For all infobases)
      • The Period-end closing date field becomes available, Other methods of specifying a period-end closing date section becomes available at the bottom of the window.
    • By users (By infobases). You can use this feature if period-end closing date is to be set for a specific user or user group (for a specific infobase) or different values of period-end closing date (data import restriction date) are to be set for different users (infobases).
      • A list, in which you can add users or user groups (infobases) and set period-end closing dates (import restriction dates) for them, becomes available.

      • The list includes the following columns:

        • User, user group(Application: infobase. Name of the application and infobase for this application).
        • Comment. Write additional information on user (infobase).
      • The list always contains the item <For all users> (<For all infobases>) by default. You cannot delete or change the item. Settings of period-end closing date (data import restriction date) used for this item will be used for all users (infobases), other than those in the list.
      • If required, add the required users or user group (infobases) to the list:
        • Click Pick to select several users (infobases) from the list.
          • If the application supports external users, you are asked to select a user type to set a period-end closing date (data import restriction date).
        • Click More actions - Add above the list toSelecta user or a user group (an application or an infobase for this application. You can also select <All infobases> for each application).
      • To set up period-end closing date (data import restriction date), in the list, select a required user, user group (infobase), or item <For all users> (<For all infobases>) and make the setup below in fields Period-end closing date indication method and Period-end closing date.

Indication methods of period-end closing date (data import restriction date)

To set up, click Other methods of specifying a period-end closing date and specify the required value in the Specify period-end closing date field:

  • Common date. One period-end closing date is specified for all items of the application.
  • By sections. Period-end closing date is set for each application section.
    • When a period-end closing date is specified by sections, a list becomes available. You can add Sections (using a corresponding command) to the list and specify a value of period-end closing date for each section. The list includes the following columns:
      • Section, object. Application sections are listed in the column.
      • Period-end closing date. Select a value type of period-end closing date from the list (you can select an arbitrary date or one of the relative dates).
      • Valid until. Write or select a value for an arbitrary date, for a relative date the value is set automatically by the application.
    • The list contains the <Common date> item by default. Value of period-end closing date for this item will be used for all sections by default, other than for those already in the list.
  • By objects. Period-end closing dates are specified by the application objects. Period-end closing date is set for each object. If you select this value, a list becomes available. You can add objects (using a corresponding command) to the list and specify a separate value of period-end closing date for each object. The <Common date> item is also available in the list.
  • By sections and objects. Period-end closing dates are set both by sections and objects.
    • The list with sections and objects becomes available. You can specify a separate value of period-end closing date for each section and object. The <Common date> item is also available in the list.
      • To select several objects in the selected section, click Pick.
      • You can also right-click the section, click Add, and then specify the object by clicking Select. You can also use the same command in the More actions menu.

To set a period-end closing date of a required section (object) or change a common date, right-click or double-click the required row of the Period-end closing date column, and then click Change. To specify a period-end closing date, click Select. When the date is selected, click OK to return to the setup procedure.

To cancel the period-end closing date, click Disable.

Period-end closing date

  • When you specify a period-end closing date, you can specify not only an arbitrary date (in this case, you can type the date or select it from the calendar by clicking ) but also a relative date.
  • To set a relative date, click More options.
  • Relative date values:
    • Last year end
    • Last quarter end
    • Last month end
    • Last week end
    • Previous day
  • For relative dates (other than the Previous day value) you can specify a number of delay days before setting a period-end closing date. Within the delay period, you will be able to change data.
    • To allow changing data within the delay period, select the Allow changing data before period-end closing date check box. Then specify a number of delay days, for example, within 10 days after the period-end closing date.

Reports on period-end closing dates

You can access reports by clicking the matching command. Reports support data grouping:

  • By sections/objects: Select the Period-end closing dates by sections and objects for users report (Period-end closing dates by sections and objects for infobases).
  • By users (infobases): Select report Period-end closing dates by users (Period-end closing dates by infobases).

Reports show the resulting period-end closing dates set for users or user groups (infobases) considering priorities.

This list is used to view object change history and choose between stored versions. Version history quickly answers the following questions: "Who changed the object?", "When were changes made?", and "What was changed in the object?".

This is a useful tool for document or list change history management, object version viewing, and object version comparison.

Before you can start working with object change history, you need to perform the initial object versioning setup. Only the application administrator can view or set up version history.

To view version history for a list or a document, use the Change history command located on its navigation panel. This command is available only to the application administrator, and only for the objects that support versioning.

The list includes the following columns:

  • # - version number. Versions are listed in descending order, with version 1 listed last;
  • Change author - name of user that made changes to the object;
  • Date - object change date and time;
  • Comment - add detailed information about this version (this feature is available to users with full access). For example, this can help analyse changes in a document. This field is filled in automatically. For example, after a version upgrade, new version number and upgrade date are added to Comment.

Versions filtering

  • Use Changes in attributes filter to show changes in certain attributes. To do this, click Select to open the list of attributes and select check boxes of the attributes whose changes you want to display in the version list.

Viewing object modification events

  • Upon adding and recording changes, you can use the Technical details on the object change link to view the event log filtered by events associated with this object.

Viewing versions

  • Click Open version to view the report describing object status on a given date.

Comparing versions

  • Select two or more versions in the object version list and click Compare versions to view the report describing object changes between the selected versions.

Version rollback

  • To roll an object back to an earlier version, select a version and click Go to version. Any changes introduced after this version was saved will be canceled.

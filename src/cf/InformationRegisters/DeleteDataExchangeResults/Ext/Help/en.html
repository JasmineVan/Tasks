This form is used to review statistics based on data synchronization results.

Data synchronization results are available as a reference in the Data synchronization list, which can be accessed from the application desktop or other locations. Depending on the results, the reference has either the No warnings value (in case of successful data synchronization) or the Warnings value (with the number of warnings in brackets). To open the list of data synchronization result, click the reference.

During data synchronization, problems may occur that require user intervention. The list contains description of data that caused synchronization problems and the causes of there problems. The causes are briefly described in a separate column; click any cause to display detailed information at the bottom of the list, including the names of list fields or document fields where problems occurred.

Filters

For your convenience, the following filters can be enabled:

  • Select the Period for which to issue data.
  • In the From field, Select a specific infobase from the list.
  • In the Cause field, type in a specific error cause (this filter is not available on the Conflicts tab).

All data synchronization problems can be roughly broken down into the following categories (each category being displayed on a separate list tab). In the header of each tab, the number of the following problems is indicated in brackets:

  • Unposted documents

  • This list displays the documents that could not be posted for any reason. To fix a document posting problem, we recommend that you open the document and eliminate any inconsistencies that prevent posting (for example, insufficient product balance). The cause that prevents document posting is displayed at the bottom of the list.
  • To start fixing errors in the selected documents, click Change selected.
  • Once all errors are fixed, Post the document again.
  • To open the document, click More - Change.
  • To hide the document from the list without fixing the posting problems, click Ignore.
  • To display the ignored documents, click More - Show ignored.
  • To return the ignored document to the list, click More - Do not ignore.

Blank attributes

  • This list displays blank attributes of catalog items. To fix this problem, open the related catalog item by double-clicking it and fill in all required fields. Then save the item.
  • To fill in the attributes of selected items, click Change selected.
  • To open the item, click More - Change.
  • To delete the catalog item from the list without elimination of filling in the mandatory attributes, click Ignore.
  • To display the ignored items, click More - Show ignored.
  • To return the ignored item to the list, click More - Do not ignore.

Conflicts

  • When data synchronization between two applications is enabled, the same data may change in both applications nearly simultaneously (between two data synchronization sessions). In this case, the next data synchronization session will encounter two different versions of the same data. This is called a data change conflict. The
    Data change conflicts are resolved automatically by prioritizing one of the two available data versions; however, you can override the conflict resolution manually. For this, select a list item.
  • Description of the conflict resolution automatically chosen by the application will be displayed at the bottom of the list.
  • To review differences between two data versions, click Show differences.
  • Before you decide which data version should be prioritized, you may want to review both versions. To do that, click More - Show version created in this infobase or More - Show version received from another infobase.
  • To view the list item, click More - Change data.
  • Once you decide which data version to preserve, click Confirm to save the result of the automatic resolution of the conflict, or click Revise to change the conflict resolution to the opposite. After that, the item becomes inactive, it is assigned the Reviewed flag, and it is hidden from the list by default.
  • All resolved conflicts are hidden from the conflict list by default. To display these conflicts, click More - Show resolved conflicts.
  • To re-review a resolved conflict and make changes to it, click More - Remove the Reviewed flag.

Items rejected due to restriction date

  • This list displays data rejected because they belong to a date period for which any data import is restricted.
  • To compare the declined data with the current application data, click Show differences (provided that data is available in the current application).
  • To open the item, click More - Change.
  • To review the other data version, click More - Show rejected version.
  • To import data regardless of any data import restrictions, click Accept version.
  • To delete the row from the list without making any decision on it, click Ignore.
  • To display the ignored items, click More - Show ignored.
  • To return the ignored item to the list, click More - Do not ignore.

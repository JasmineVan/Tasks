Use this window for task operations. This list contains all tasks.

To open the list, click All tasks in the Business processes and tasks section.

Tasks not assigned for execution are marked in bold. Overdue tasks are highlighted in red.

The list includes the following columns:

  • Importance. Takes Normal (default), Low, High values. Use this field to specify that execution of a business process can be postponed or, to the contrary, should be accelerated.
  • Task. Full description of the task. Completed tasks are marked with the icon.
  • Number. Assigned automatically.
  • Task Assignee.
  • Stopped business processes are marked with icon in this column.
  • Task Author.
  • Written. The date and time of task creation. To have time displayed for the dates in this list, select the Date and time in task deadlines check box in the Administration – Organizer section.
  • Due date. Date and time by which the task is to be completed.
  • Completed. The date and time the task was actually completed.

Show

  • Use the Show field for your convenience to display:
    • All tasks
    • Completed tasks
    • Not completed tasks

Filters

The following filters are available for the task list:

  • By Author. Select the author from the Users list.
  • By Assignee. Select the assignee from the Users list.
  • By period. Click More – Set date interval, or right-click the task and select the corresponding command.

Viewing a task

  • To view a task, double-click it,
  • or click More – Change, or right-click the task and click Change.

Viewing a subject

  • Tasks can be created based on application objects. To view the task subject, execute the Open task subject command of the context menu by right-clicking, or find the command in the More menu.

Viewing a business process

  • Each task is a step of a business process. To view the business process a task belongs to, click More – Open business process, or right-click the task and execute the corresponding command.

Accepting for execution

  • Tasks not yet accepted are marked in bold.
  • Use the matching button to accept a task For execution. In the case of role addressing, the task accepted for execution disappears from the tasks lists of other performers who are assigned to the same role.
  • In order to refuse to execute the previously accepted task, you must use the context menu command by right-clicking, or find the command in the More - Cancel accepting the task for execution menu.

Execution

  • Use the context menu command by right-clicking, or find the command in the More - Completed menu to mark the task as completed.
  • Completed tasks are mark with the icon in the list.

Forward to another performer

  • Click Forward, fill data about another performer.

Subordinate business processes

  • New business processes (such as Job) can be created on the basis of tasks. To create a business process on the basis of a task, click Generate based on. This requires sufficient rights. To enable creation of business processes on the basis of tasks, select the Subordinate business processes check box in Administration – Organizer section.

Reminder

  • To remember to execute the task, you can set a Reminder, the text of which will be displayed at a specified time.
  • Click , select an action, for example, Remind, fill the required fields.

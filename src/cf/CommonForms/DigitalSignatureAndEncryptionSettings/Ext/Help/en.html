Manages parameters for digital signature and encryption utilities.

To open this form, click Digital signature and encryption settings in Administration - Common settings section.

To use the digital signature and encryption functionality, you need to have a third-party crypto provider application installed.

To enable data signing, select the Digital signature check box, and to enable data encryption, select the Encryption check box in Administration - Common settings section. This enables the corresponding settings and commands.

Digital signature verifies that an electronic document was not altered in any way since the moment it was signed. To apply a digital signature, you need a digital signature key certificate. Certificates are issued by certification authorities. For example, you can apply for a certificate to the 1С ceritification authority athttp://ca.1c.ru/. Legally valid (qualified) certificates are only available from the certification authorities.

For distributed infobases, the majority of settings can only be modified from the master node.

Certificates

  • Displays all certificates used to sign and encrypt data, and all certificate issue applications.
  • To show only your certificates, enable filter in the Show field. By default, all certificates are displayed.
  • You can also use filter by the Application state field ("Any", "Not prepared", "Prepared", "Submitted", "Rejected", "Committed, certificate not installed", "Committed").
  • You can Add a Certificate from the list of certificates installed on computer. This can also be done during data signing or encryption.
  • To create an Application for new certificate issue, click Add.
  • To view a certificate or a certificate issue application, double-click it. Or, use the Edit command available in the context menu and the More menu.

Applications

  • Contains a list of digital signing and encryption applications to be used in the infobase (including all nodes of distributed infobases).
  • Installing these applications on all computers is not required. Moreover, installing several application from this list on a computer may result in software conflicts.
  • By default, the list contains the applications most popular in Russia: ViPNet CSP and CryptoPro CSP.
  • You can add other applications to the list easily (predefined settings for Microsoft Enhanced CSP, ViPNet CSP, CryptoPro CSP, LISSI CSP, Signal-COM CSP are available).
  • To add an encryption application that has no predefined settings, specify its name, type, and algorithms, in accordance with the application manual.
  • The Check result column displays messages that describe whether the application is installed locally and whether the installation is valid.
  • Linux: Path to application (<Name of application corresponding the selected list item>) - this field is mandatory for computers running under Linux.
  • Below the list, the How to work with digital signature and encryption applications hyperlink is located. If you have any questions, click this link to access the help section.
  • Check signatures and certificates on server check box is available for server-based infobases only. If it is selected, a digital signature and a certificate are checked on the server. In this case, it is not required to install the application on every computer, on which only such actions are expected.
    • From the application window, you can access the Application paths on Linux servers list. In this list, you need to specify paths for the computers running 1C:Enterprise servers under Linux.

More

  • Specify Encrypted file extension when sending files by email, when executing the Save as command, and when exporting folders. The default extension is .p7m.
  • Specify Signature file extension (the default extension is .p7s).
  • Use the On saving data to the file with digital signature switch to define how the application behaves when the Save with digital signature command is executed:
    • Save all signatures - all signatures are saved
    • Prompt - the user selects the signatures to save from list
  • To save the certificate together with the file and digital signature, select the Save certificate check box.

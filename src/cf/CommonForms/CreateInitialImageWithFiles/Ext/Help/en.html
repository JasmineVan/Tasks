This form is used to specify File mode or Client/server mode infobase parameters when creating initial images for distributed infobases (DIB). When setting up DIB synchronization (external files), you need to specify the location where the initial DIB image will be created. In case of volume-based file storage, you will also need to specify path to the archive (.zip) file containing the compressed files stored in volumes.

It is accessible via command Create initial image located in Data synchronization creation wizard during DIB synchronization setup (external files). Alternately, in the Data synchronization list, click to open DIB synchronization setup (external files) and click Create initial image of subordinate DIB node.

Infobase parameters

Select the location for the initial infobase image via radio button:

  • On this computer or on a computer in the local network.
    • Specify Infobase parameters:
    • Language (Country) – for this field, select a value from list. By default, parameters specified during data synchronization creation are used.
    • Specify Full name of file infobase:
      • For 1C:Enterprise servers under Microsoft Windows Specify path to the folder where the file will be stored by clicking .

or

      • For 1C:Enterprise servers under Linux – specify path to the folder where the initial image file will be stored.
  • On 1C:Enterprise server.
    • Specify Infobase parameters:
      • Specify 1C:Enterprise server cluster.
      • Type in Name of infobase in cluster.
      • In the Secure connection field, select a value from the list (Off, Establishing connection, or Always on).
      • Select DBMS type from the list (Microsoft SQL Server, PostgreSQL, IBM DB2, or Oracle Database).
      • Specify Database server.
      • Type in Database name.
      • Type in Database user name and User Password.
      • Select Date offset (0 by default).
      • Language (Country) – select a value from the list. By default, parameters specified during data synchronization creation are used.
      • Select the Set scheduled job lock check box if necessary (off by default).

Path to volume file archives

  • Specify Full name of volume file archive:
    • For 1C:Enterprise servers under Microsoft Windows – specify path to the directory by clicking

or

    • For 1C:Enterprise servers under Linux – specify path to the folder where the volume file archive will be stored.

Click Create initial image to proceed.

After a pause, you will be informed that initial image of subordinate DIB node was created successfully. Click OK to go back to the application.

To load the volume files to local branch infobase when starting the application for the first time, you will need to execute the Store initial image files and specify the path to the volume file archive.

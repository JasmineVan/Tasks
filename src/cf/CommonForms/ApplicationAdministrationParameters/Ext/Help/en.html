This form is used to specify server cluster access parameters.

It opens from the Application lock window when you click Enable scheduled job lock management.

The following parameters are required to perform a variety of administrative activities with a client/server mode infobase:

  • Name and password for Infobase administrator required to lock scheduled and background jobs on their behalf. Generally, you can use your own user name and password (the same name and password you type in every time when logging on). However, for security purposes, we recommend that you create a separate technical user and grant them administrative rights. Keep in mind that user name and password are directly stored in the infobase and can be compromised if a malicious user gains full infobase access.
  • Cluster administration parameters:
    • Name and password for any Server cluster administrator on whose behalf user sessions are terminated. Do not set this parameter if no server cluster administrators are available.
    • Port numbers for accessing server cluster and server agent (provided that default port numbers cannot be used). Generally, ports for server agent and for server cluster are numbered sequentially, for example: 3040 and 3041.
    • Connection to server cluster. Select an option from the list:
      • COM connection to server agent
      • Administration server (ras) connection
    • Address, for example, "demo".
    • Port, for example, "3040".

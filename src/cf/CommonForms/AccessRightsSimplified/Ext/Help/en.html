This form is used for simplified setting of access rights.

User access rights management is profile-based. A profile defines a set of actions with the application data the user is allowed to perform. Generally, profiles correspond to user job functions or duties. Multiple profiles can be assigned to the user at the same time, these profiles collectively defining their personal access rights settings.

To set up user (external user) access rights, execute the Access rights command located on the navigation panel.

Profiles

  • In the Profiles list, select check boxes for the profiles required for the user's application activities.
  • Generally, a profile incorporates several roles. Assigning a profile to user automatically assigns them all roles contained in the profile. For example, the Administrator profile contains the Full rights role granting unlimited access to all data. Only application administrators (users with the Administrator profile) can set up user access rights.
  • All profiles are broken down into two categories:
    • Primary profiles define a set of access rights sufficient to perform specific activities in the application.
    • Using the additional profiles, a user can be given auxiliary rights in addition to the main ones. Access kinds
  • This list is displayed to the right of the profile list, provided that the Use record-level security check box is selected in the Administration - User and rights settings section. The Access kind field is used to precisely define the data area accessible to the user.
  • Access kind is a some kind of rule that allows access to the application data. For example, using the Companies access kind, you can allow users to work only with documents on specific companies, or vice versa, hide documents of some companies from them.
    • Contents of the access kind list may vary depending on the profile selected in the profile list (to the left). In addition, the profile also determines how the access kind setting works: whether it allows only to view or also to edit documents. For example, if a profile includes the Goods receipt document read role, users with this profile are only allowed to view these documents. If the profile contains a role of adding and editing, then it becomes possible to edit such documents.
  • You need to specify Access values for each access kind. Select All prohibited or All allowed from the list, or specify exceptions.
    • Any access kind can be set up by two methods:
      • Specify the allowed value list. We recommend that you use this setup method when you need to grant the users access to a specific limited data area.
      • Specify the prohibited value list.
        • This setting method is convenient in cases when you need to lock access to a previously known data area, and to all new values entered into the program, the allowing rules should be applied by default.
        • For example, if using the Companies access kind the right is set that prohibits to work with the documents by the Forward Thinking company, these restriction will not be applied to the new company New Tech, added to the application later.
        • Please keep in mind that "allow" settings have priority over "prohibit" settings. If in the example above, the user is allowed to enter documents on the Forward Thinking company in some other group with the same profile, then in the end the user will get access to all documents on this company.
    • To create exceptions, click Add and select the required values from list.
    • In some cases, to simplify the setting of access rights, you can specify not individual elements, but the entire groups of elements in the list of allowed or prohibited values.
      • For example, if you set up the Partners access kind to prohibit access to some of the partners, you can specify partner access groups instead of individual partners. For this, you will need to create the list of partner access groups, and assign each partner to a group.
        • These activities are not limited to administrators; any user with sufficient rights to edit the partner list can perform this.

The Users access kind

  • For some profiles, the Users (External users) access kind can be used for access rights setup.
  • This applies only to the documents (and other application objects) containing fields that can be used to specify users allowed to access these documents (such as Person responsible, Author, and Performer).
  • To grant access to all restricted data available to a specific user, you need to add this user to the allowed value list for the Users access kind.
  • For example, if access to goods and service receipt documents is governed by the Responsible person field value, then a user will be able to save documents where Responsible person matches the user or added to the list of allowed for the Users access kind.
  • The prohibiting setting acts in the same way.

Access rights report

  • To view the full list of user access rights, including the full list of applied objects and access rights per object granted to this user (user group), User access rights report can be generated.
  • Click Access rights report.

View unused access kinds

  • To view those access kinds, that are not used at the moment (for example, this feature is disabled in the application), click More - Show unused access kinds.

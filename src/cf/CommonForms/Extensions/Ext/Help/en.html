Use this list to add configuration extensions to the application. Configuration extensions add new functionality without changing the application directly.

For more information about extensions, see Configuration extensions.

To open this list, in section Administration—Print forms, reports, and data processors, click Extensions.

The list includes the following columns:

  • The check box column. Select or clear the check box to attach or detach available extensions.
    • The application displays a warning before detaching an extension. Detaching an extension might make application data stored in the extension inaccessible. You will be unable to change such data after detaching the extension. To confirm the detachment, click Detach.
  • Description. This field is filled automatically when you attach an extension. It is specified by the extension developer.
  • Version. This field is filled automatically when you attach an extension. It is specified by the extension developer.
  • Purpose. This field is filled automatically when you attach an extension. It is specified by the extension developer.
  • Send to subordinate nodes of the distributed infobase. If this check box is selected, extensions are copied to the subordinate nodes of the distributed infobase. It is cleared by default.
  • Safe mode. When you attach an extension, this check box is selected. An administrator can clear it or specify the security profile name.

The extensions status icons are also displayed in the list. The following icons are available:

  • Attached (a green circle). For more information about attaching extensions, see http://its.1c.ru/db/v837doc#bookmark:dev:TI000001534 (in Russian).
  • Not attached (a red circle).
  • Installed (a gray circle). Changes status to "attached" or "not attached" after restarting the application.

Adding extensions

  • Click Add from file.
  • The application displays a warning before installing the extension. If the extension is received from a secure source, click Continue.
  • Specify the path to the extension file.
  • Adding an extension will take some time.
  • Once an extension is added, click restart for the change to take effect. This link is not available in internet applications as extensions are not attached when shared users (for example, service administrators) enter the data area. Instead, such users retain the set of extensions that was available before they entered the data area (since the session was started).
  • After the restart, the extension becomes attached to the application and ready for use.

Deleting extensions

  • If you no longer need an extension, select it and click Delete.
  • Deleting an extension does not detach it. All sessions started after the extension is added and before it is deleted still use the deleted extension.
  • For new sessions, the deleted extension is not attached.

Restarting the application

  • To apply the changes made to the extensions list, click restart.
  • If an extension cannot be attached to the application at the startup, a warning is displayed.

Saving extensions to files

  • Right-click an extension, click Save as, and specify the file.

Updating extensions

  • Select an extension, click Update from file, and select a file.
  • Use this button to attach new extension versions.
  • You can also use this button to switch back to the previous version if the new version contains errors. However, it is not recommended as it can clear the settings.

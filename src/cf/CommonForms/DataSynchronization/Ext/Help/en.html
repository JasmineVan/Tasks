This form is used to set up data synchronization and store the data synchronization settings.

The Data synchronization settings list can be accessed from the application Desktop or from the Administration - Data synchronization section, by executing the related command.

The window displays the following information:

  • Application - description of the application participating in this data synchronization;
  • Correspondent infobase prefix;
  • Data synchronization status;

  • Information on dates of executed data sending and receiving. Unsuccessful data sending and receiving are marked with the red icon . Warnings are marked with . Synchronization status (minutes elapsed since the last synchronization attempt, Now, Never) is also displayed here:

    • Data sent - time the latest data was sent;

    • Data received - time the latest data was received.
  • - data synchronization is performed on schedule. Data synchronizations automatically performed under a predefined scenario are also marked with this icon in this column;
  • Prefix of this infobase is displayed at the bottom of the list.

Data synchronization statuses

  • Data synchronization statuses are displayed in the list in the related column:
    • Last start - synchronization is configured successfully and is running. Data synchronization start date is specified;
    • Not started - synchronization is configured successfully, but data exchange has not been performed yet. In this case, click the Start synchronization button;
    • Setup not completed - synchronization setup was interrupted. In this case, you can continue setup by clicking the Set up button
    • Setup not complete, data for mapping received - this data synchronization setup status is displayed in the destination application in case of direct connection. Further configuration is required.
    • Data for mapping received - to analyze and map received data, click Start synchronization.

Data synchronization configuration

  • To set up a new data synchronization with an external application, execute the related command from the New data synchronization submenu. Data synchronization setup window will be displayed.

Viewing setup information

  • For more information on the accounting specifics in applications participating in data synchronization, and for recommendations on data synchronization setup, click More - More on data synchronization with this application.

Data Sync

  • Select a setting, click Start synchronization to start data synchronization.
  • To synchronize data between two applications, identical data mapping is run. This method is useful when identical data was independently entered into both applications for any reason. To reduce this risk, you can set up frequent data synchronization on schedule.

Changing data synchronization

  • To change or continue data synchronization setup, click Set up
  • You can also double-click the item.

Viewing sent data content

Data export for mapping

  • Execute the More - Data export for mapping menu command.

Data synchronization on schedule

  • To Set up data synchronization on the specified schedule, execute the More - Data synchronization scenarios command. In this case, data synchronization in client/server mode will run in the background, in the form of Data synchronization background job. However, when in file mode, you will need to start a separate session to perform background jobs.
  • You can also access this command by right-clicking to open the context menu.

Viewing warnings

  • Click Warnings (number of warnings) for detailed information on the warnings that occurred during data synchronization.

Viewing event log

  • Detailed information on data synchronization execution is stored in the application event log. To view the list of data synchronization events or events themselves, execute the More - Send data events or Receive data events command.

Replacing synchronization rules using an external file

  • To use external rule files instead of standard object conversion and object change record rules, you will need to:
    • Open data synchronization for which you want to use the external rules.
    • Execute the More - Import rules command.
    • In the object conversion or object change record window, set the radio button to Import from file on computer and click Import to locate a rule file stored on the local computer or in a network folder.
    • Make sure that the imported files are correct and suitable for the source and destination applications in use.
    • Click Write and close. The imported rules will apply immediately.

Deleting data synchronization settings

  • Select a setting and click More - Delete synchronization setting.
  • Warning! This will delete the settings irrevocably.

Restoring infobases from backup

  • When restoring infobases from backup, we recommend that you:
    • Temporary disable automatic synchronization (clear the Automatically on schedule check box in all data synchronization scenarios);
    • Cancel sending data to other applications for synchronization purposes (for each data synchronization settings, you will need to execute the More - Sent data content command, then select the object tree root in the Recording changes windows, and execute the Cancel recording - All objects of selected types command);
    • Re-import data that was entered into other applications since the creation of this infobase backup copy;
    • Enable the automated data synchronization.
  • If a significant data volume was lost, we recommend that you delete all data synchronization settings and set up the data synchronization once more. Otherwise, other applications participating in data synchronization may receive the outdated data from the restored infobase, resulting in data loss.

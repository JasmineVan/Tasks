Use this form to restore the connection between a subordinate node and a master node.

The message is displayed when you sign in a subordinate node that is temporarily detached from a master node (after canceling connection with the master node).

The following cases require canceling connection with a master node (temporary detachment of a subordinate node):

  • Importing the master node configuration to the subordinate node while restoring a distributed infobase (you must not import an older configuration version).
  • Restoring the master node configuration in case of failure that resulted in the master node infobase corruption.

To attach the infobase back to the master node before resuming the startup, click Attach.

To irrevocably detach the infobase from the master node before resuming the startup, click Detach. It is required after canceling connection with the master node in the following cases:

  • creating a master node from a copy of a subordinate node;
  • creating a standalone infobase from a subordinate node (or its copy).
  • Warning! This operation is irreversible. After that, the connection to the master node can be restored only programmatically.

Use this window to set authorization parameters for internal or external users.

Password requirements

To prevent unauthorized access to the application, select a combination of the following check boxes:

  • Apply password complexity requirements. If you select this check box, the application will accept only passwords that are at least 7 characters long; contain any 3 of the following 4 character types: uppercase letters, lowercase letters, numbers, and special characters; and are not identical to usernames.
  • Minimum password length. The default value is 8 characters.
  • Maximum password validity period. The period (in days) that a password can be used after the first authorization and before the application requires the user to change it. The default value is 30 days.
  • Minimum password validity period. The period (in days) that a password must be used before the user can change it. The default value is 1 day.
  • Deny reusing recent passwords. The number of unique new passwords that have to be associated with a user account before an old password can be reused. The default value is 10. Selecting this option enhances security by ensuring that old passwords are not reused continually.
  • Inactivity period before denying authorization. The period (in days) after the last authorization after which the authorization is denied. The default value is 45 days.

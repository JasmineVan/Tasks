This chapter describes how to set up proxy server parameters.

The application allows you to get files from the Internet by the HTTP, HTTPS, and FTP protocols and save the received files on the client, on the server, or in the temporary storage.

You might need access to the Internet to perform certain operations, e.g. download exchange rates or the address classifier, exchange data via an FTP resource, update the configuration from the vendor website, etc.

In case you access the Internet via a proxy server, specify proxy server parameters in the application. There are client and server proxy server settings.

Generally, proxy server parameters are required for mechanisms that get files from the Internet by the HTTP, HTTPS, and FTP protocols. Server settings are required for the mechanisms to work on the server.

As for the client mechanisms, e.g. downloading exchange rates in the thin or thick client, you need to customize the workplace. That said, you do not need to set up the proxy server in order to work with emails. Proxy server parameters are usually specified for each workplace, because several workplaces might have different settings for Internet access.

If company uses client-server 1C:Enterprise version, proxy server parameters must be specified on the 1C:Enterprise server(from the Administration. Common settings. Server infobase parameters section). Proxy server parameters for workplace and 1C:Enterprise server can differ.

To access the Internet from client computers, the proxy server parameters must be specified on each client workstation (via Personal settings of the Administration section).

Warning: if security profiles are enabled, you must use the inetcfg.xml configuration file to set up the proxy server parameters.

Setting up proxy server parameters

  • Select a proxy server usage option:
    • Do not use proxy server. Default value upon the first start of the application.
    • Use system proxy settings. The settings are automatically filled in by the proxy server settings of the operating system. In this case, the filled in settings are not available for editing.
    • Use other proxy settings. The settings are specified manually:
      • Type the Server address and the server Port (the parameter is the same for the 1C:Enterprise server and for the workplaces).
      • From the drop-down list, select an Authentication method:
        • Base. In this case, specify proxy server user and their password for each workplace and the server (usually different, it depends on the company's network policy).
        • Operating system. In this case, the account of an operating system user will be used to work with the proxy server.
      • Specify the name of the User and the Password.
      • If you select the Bypass proxy for local URLs check box for each workplace and the server, you will be able to decrease the proxy server load in the company's local network.
  • To confirm proxy server parameters, click OK.

Additional parameters

  • Click More:
    • to view additional parameters if you use proxy server system settings.
    • to go to the additional parameters customization if you use other proxy server settings.

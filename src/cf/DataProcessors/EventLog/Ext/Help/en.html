Application administrators might want to check which events occurred in the application at specific time or which actions were performed by a specific user. This data is available in the event log.

The event log stores important user activities, such as data modification, execution of scheduled operations, and signing in or out. You can configure the event log in Designer.

To open the event log, click Administration—Support and service.

All application events are grouped into the following types and marked in event log correspondingly:

Error.
Warning.
Information.
Information on unavailable fields.
Note.
Each log line represents a single event.

The list includes the following columns:

  • Date, time. The event date and time, and the icon that indicates the event type.
  • User. The full name of a user.
  • Computer. The network name of a computer.
  • Application. For example, Thin client.
  • Session. A session number.
  • Event. An event type.
  • Data, Comment. An application object, such as a particular list item or document, and a brief description of the event.
  • Metadata. An application object name and type, such as list or document type.

To configure the number or columns, their order, and visibility, click More actions—Change form.

Exporting event log to a file

  • Click Export to file and select a directory. The log is exported to an XML file. You can open it with Microsoft Excel and similar applications.

Refreshing event list

  • The list of events is not updated automatically. To view the events that occurred after the event log was opened, click Refresh or press F5.

Number of events

  • The event log displays a fixed number of records. It is specified in the Events to display field. The default value is 200. You can select a predefined value from the list (200, 500, 1000, or 10000 events) or enter a custom value.

Filters

You can apply filters to the event log.

  • To display the events of a specific type, in the Severityquick filter list, select Errors or Warnings. By default, the log displays All events.
  • To view events for a specific period, click Period. or double-click the Date, time column.
  • Select data in a log column and click Filter by current column value. The event log will display only events containing the selected data. This command is also available in the context menu.
  • To filter events by one or several criteria, click Filter or follow the link in the Filter field. The filter link changes text based on the filter criteria (including quick filters). If no filter is applied, the link text is Not set. The following filters are available:
    • In the Period field, select a period or fill in the from and to fields.
    • In the Severity field, select or clear the Error, Warning, Information, and Note check boxes. By default, all of the check boxes are selected.
    • In the Events list, select or clear the check boxes next to event types. By default, all of the check boxes are selected.
    • In the Users list, select or clear the check boxes next to users. This list displays only the users that appear in the event log. By default, all of the check boxes are cleared.
    • In the Applications list, select or clear the check boxes next to the application types (Designer, Thin client, and Background job). By default, all of the check boxes are cleared.
    • In the Computers field, select or clear the check boxes next to computers. By default, all of the check boxes are cleared.
    • In the Sessions field, enter session numbers separated by ;.
    • In the Data group, select one of the following filter types:
      • In the Metadata field, select or clear the check boxes next to metadata types. For instance, the "Bank classifier" list.
      • In the Data field, you can select specific list items.
        • First, select a Data type(for instance, "Bank").
        • Then select a specific item. For example, in the "Bank classifier" list, select a bank.
    • In the Data presentation field, enter or paste data to filter by.
    • In the Transaction group, select or clear the Transaction status check boxes No transaction, Committed, Not completed, and Rolled back. By default, all of the check boxes are selected. You can also filter data by transaction ID.
    • In the Other properties group, you can filter data by selecting the Working servers, Main IP ports, and Additional IP ports check boxes. You can also filter data by Comment.
  • To apply the filter, click OK. The filter link changes to the filter description.
  • The number of the displayed records is limited by the value of the Show field.
  • To clear all filters (including quick filters), click Clear filter. You can also use the More actions—Clear filter command. This command is also available in the context menu.

Viewing events

  • To view event details, select it in the list and click Open. This command is also available in the context menu.
  • You can also double-click an event.

Viewing application objects

If an event is related to an application object, you can view the object.

  • Select data in the Data, Comment column of the event and click View data. This command is also available in the context menu.
  • You can also view this data by double-clicking in the Data, Comment column.

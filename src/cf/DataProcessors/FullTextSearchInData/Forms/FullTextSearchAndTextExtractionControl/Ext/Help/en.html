Use this form to enable and disable full-text search, update full-text search index, and set up extraction of text from file for using upon the search.

Opened by the reference Set up Administration. Common settings. Data search section.

This application allows to perform full-text search in all directories, documents and other objects, looking for a specified string in any object field. If any attached files containing text data are found, their contents will also be searched. Full-text search is used when searching data in the in application by the contents of directories, documents and other applied objects.

The full-text search feature supports: latinization (conversion of non-Latin characters to Latin), substitution (using Latin characters instead of identically-looking non-Latin letters), and fuzzy search (finding strings approximately matching the search string).

To start working with full-text search, select the Full-text data search check box. In this case, the commands that allow you to configure full-text data search in the application, manage the full-text search index, and update and clear it, become available. The check box is enabled by default.

In the application file mode only commands that manage full-text index are available, texts are extracted automatically at importing file into the application.

Temporary disabling of full-text search

  • Check box Use full-text search is enabled by default. It is recommended that you disable the full-text search if it is necessary to improve application speed slightly and data search in application is not required to users.

Size of data being indexed

  • Using according check box, you can Restrict max size of data being indexed. Restriction is 1 MB by default. It can be in demand if the computer resources, where the application is installed, are restricted.

Update full-text search index

  • In order to search by all the data entered into the application, it is necessary to regularly update the full-text search index. To automatically update the index, the Update full text search index and Merge full text search index scheduled jobs are used (executed once in 24 hours).
  • If the index has already been updated, then the Index relevance date is displayed. It is the date of the index last update.
  • If the application has data that has not been indexed, then the Update Index button becomes available and the "Update Required" index status is displayed.

Clear full-text search index

  • If search operation is damaged for some reason, click Clear index to delete all files of the full-text index. After you delete the index, you need to build it again. To do it, click Update index.

Check full-text search index

  • You can check the status of the full-text search index at any time, clicking Check index.

In the client/server mode, commands for setting up automatic text extraction are available.

Automatic text extraction

Text extraction from files is required for full-text search in the application.

To configure, select the following using the switch:

  • Servers under OS Windows in the cluster. In this case, the Configure schedule command becomes available, using which you can configure scheduled job. Specify schedule and other parameters of scheduled job for text extraction from files.

    • This option is suitable, when the following rule for server running under Microsoft Windows is specified in the Administration of 1C Enterprize servers :
    • Rule object: Client connection with the infobase.
    • Rule type: Assign.
    • infobase name: not specified.
    • Additional parameter value: BackgroundJob.CommonModule.FullTextSearchServerUpdate.FullTextSearchIndexOnSchedule.
  • No servers under OS Windows in the cluster (all servers in the cluster are under OS Linux only). In this case, using the Start text extraction command, you can start text extraction in the thin client on a workstation under Windows.
    • If in client/server mode one or more working processes run on a Linux server, and one or more client computers are connected via web client, the infobase can contain files that skipped text extraction procedure and therefore cannot be found by using full-text content search. In this case, you need to start thin or thick client on any client computer running under Microsoft Windows, and execute the Text extraction to extract text from files in the automatic mode. The default execution time interval is 60 seconds.

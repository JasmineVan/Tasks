The assistant is used for configuring data synchronization. You can synchronize data between two applications using exchange rules, without exchange rules, or organize collaboration in the distributed infobase.

Selecting data synchronization type

Select the required list section:

  • Data exchange with other applications;
    • Not using exchange rules;
    • Full synchronization;
    • Getting data;
    • Full data synchronization via EnterpriseData. If you configure exchange using ordinary communication channels (file, FTP, email), you cannot set up sending and receiving rules in a source application unless you configure settings in a destination application.
    • Full synchronization supporting backward compatibility;
    • Data synchronization via universal format;
  • Distributed infobases.

Stages of configuring parameters of data synchronization between two applications

  • Data synchronization is configured in two stages. Stage 1 of data synchronization setup is performed in this application; then, you need to sign in to the other application to continue with the data synchronization setup. To transfer information between the data synchronization setup stages, a temporary file is used. You must set up data sending and receiving rules in each application participating in the exchange regardless of the selected connection option.
  • The following parameters are defined during the setup procedure:
    • Connection parameters. The parameters provided by user are validated;
    • Data synchronization rules;
    • Initial data export;
    • The destination application must also map and import data in addition to these steps.

Features of synchronization via EnterpriseData universal format

  • You can set up data sending and receiving rules only after getting parameters of a destination application.
  • To do that, upon configuring connection options in a source application, go to the destination application, complete the settings in it and perform initial data export, then complete the settings in the source application. The application displays a notification about this.

Detailed description

  • Click the Details hyperlink to view the detailed information on both applications participating in this data synchronization.

Backup

  • We recommend that you back up your infobases before performing data synchronization setup.
  • In infobase file mode, click the link to create data backup (recommended).
  • In server mode and other modes, back up data using third-party utilities.
  • Click Next to proceed with data synchronization setup.

Selecting connection to target application

The application automatically validates the connection option during configuration.

To use connection options requiring the Internet, click Internet access parameters, then configure proxy server parameters by clicking the appropriate link if necessary.

When entering user logon name and password, keep in mind that data synchronization success depends on this user's access rights (for example, if the user has no access to 1C:Enterprise server, data synchronization may fail).

Use radio buttons to select the following:

  • Direct connection on this computer or in a local network is used to synchronize data between two applications. In this case, both applications must be available for connection.

    • Enter the target application connection parameters:

      • Select the destination Application location:
        • On this computer or in a local network;
          • Select an infobase Directory. Specify a path to a local folder.
        • On 1C:Enterprise server;
          • Enter a Server;
          • Enter an infobase name.
      • use Application authorization parameters radio buttons to select the following:
        • Use operating system account;
        • Use specified username and password:
          • Enter Name and Password of the user on whose behalf data synchronization will be performed.
    • Click Next.
  • Connecting to an application via the Internet (web service):
    • This connection option is used to synchronize data between two applications. It can be used when the destination application is located in another office and is available for synchronizing via the Internet.
    • Enter the application URL;
    • Enter Username and Password;
    • select the Remember password check box.
    • To set up proxy server parameters if necessary, click Internet access parameters.
  • Synchronizing data via file, without connecting to another application (network directory, FTP, email);
    • This option can be used in the following cases:
    • to synchronize data with an application that is located in another office and can operate in standalone mode. In this case, the two applications participating in data synchronization can operate at different time.
      • To proceed with configuring synchronization in the destination application, select the Import connection parameters from file and specify the path to the settings file.
    • To synchronize data in a distributed infobase.
  • Synchronization connection is executed from another application. If this option is used, the application sends and receives data via the Internet only upon a request from the correspondent application.

To proceed with configuring data synchronization, click Next.

Data synchronization via local or network directory

  • Select the Use local or network directory for data synchronization check box.
  • In the Directory field, select a path to a local folder or a network location.
  • To reduce the size of the outgoing message, select the Compress sent data to archive check box. Enter Archive password if necessary.
  • To ensure the correctness of the entered settings, click Check whether it is possible to write to directory.
  • To proceed with configuring data synchronization, click Next.

Data synchronization via directory on FTP server

  • Select the Use directory on FTP server to synchronize data check box.
  • Specify Directory address. Port is specified automatically.
  • Fill in the User (Username) and Password fields.
  • If the data size on the FTP server is limited, select the File size limit check box and specify the maximum file size.
  • If necessary, select the Passive mode check box.
  • If necessary, click Internet access parameters and specify proxy server parameters.
  • To proceed with configuring data synchronization, click Next.

Data synchronization via email

  • Enable the Use email for data synchronization check box.
  • In the Account field, select an account from the list or click + Create (press F8) to add a new account.
  • To limit the size of sent messages, select the Attachment size limit check box. Specify the maximum attachment size.
  • To proceed with configuring data synchronization, click Next.

Additional settings for synchronization via file

  • Select the Transliterate file names of exchange messages check box. File names will be automatically written with Latin characters.
  • To reduce the size of the outgoing message, select the Archive exchange files check box. Enter Archive password if necessary.

    Features of data synchronization via universal format or EnterpriseData format via file

    • When configuring synchronization via EnterpriseData format or via universal format, the composition and other characteristics of data to be received are set up with sending rules in the correspondent application.
    • Connection option configuration differs:
    • Set up a connection option via file in a source application;
    • Continue data synchronization setup in a destination application. Configure connection option via file.
    • To do this, go to a destination application, select Data synchronization via file as a connection option, and specify the settings file Data synchronization via universal format.xml (received from the source application) without connecting to another application.
    • You can proceed with further data synchronization configuration only after setting up connection options in both applications.

    Configuring data sending and receiving rules

    If you must synchronize only part of data (not all data), use data synchronization rules.

    • Use the Documents radio buttons to define the way of synchronization:
      • Send and receive automatically - documents will be synchronized in both applications;
      • Send automatically - documents will be automatically sent to the second application. The first application will not automatically receive documents from the second application;
      • Receive automatically - documents will be received from the second application. Documents will not be sent by the first application;
      • Send and receive manually - documents will not be synchronized automatically. You must send and receive documents manually.
    • Use the Regulatory and legislation data radio buttons to select the application's list export mode:
      • Send and receive automatically - data will be synchronized in full. Regulatory and legislation data will be automatically synchronized in both applications;
      • Send and receive only information used in documents - data will be synchronized in full. Regulatory and legislation data will be automatically synchronized in both applications;
      • Send and receive manually - regulatory and legislation data will not be automatically synchronized. You must send and receive this information manually.
    • Specify the Date of document sending start. The default value is the beginning of the current year.
    • You can limit data participating in synchronization and specify sending data filters broken down by selected Companies, Warehouses, and Departments:
      • The By companies filter limits the list of companies for which data synchronization is allowed. The enabled filter affects sending the companies list and other data related to companies (lists and documents). To enable the filter, select the By selected (companies) radio button and use check boxes to specify the required companies from the list.
      • To enable the By departments filter, on the appropriate tab, select the By selected (departments) radio button and use check boxes to specify the required departments.
      • You can also export information By selected warehouses. To do this, on the appropriate tab, select the By selected radio button and use check boxes to specify the required warehouses.
    • Data receiving rules - default values are required for correct document recording. The rules are used for automatic filling data in documents that cannot be received from exchange. For example, some documents received from another application do not contain certain attributes. In this case, the attributes will be automatically filled in by the application during data synchronization. You can specify these attributes while configuring synchronization (click Change autofill values).

    Data synchronization parameters

    • Presentation of this application is outputted:
      • Description.
      • Prefix;
    • Presentation of the correspondent application is outputted:
      • Description.
      • Prefix.
    • Click Next to check the provided parameters and complete data synchronization setup for this application (in case of direct connection between applications, you will need to set up and validate parameters for the other application as well).

    Switching to interactive data synchronization

    • There is a high risk of data duplication when synchronizing data between two applications. To map data, it is recommended to use Interactive data synchronization wizard.

    Specifics of data synchronization setup in client/server mode

    • When setting up data synchronization in client/server infobase mode, keep in mind that all data will be synchronized on a computer where 1C:Enterprise server is installed. The path to the 1C:Enterprise server running on Microsoft Windows will look like this:
      \\servername\resource
      . The path to the server running on Linux will look like this:
      //servername/resource.
    • Make sure that the user on whose behalf the 1C:Enterprise server runs has sufficient rights to read and write in the data synchronization folder. If the user does not have the sufficient rights, you will need to grant them these rights, or select another user with sufficient rights to read and write in the data synchronization folder and run the 1C:Enterprise server on their behalf. The same recommendations apply to FTP and email access. For more information on 1C:Enterprise server administration, see the 1C:Enterprise platform documentation.

    Specifics of distributed infobase node creation

    During Stage 1, the initial distributed infobase (DIB) image is created. All wizard pages remain as they are. On the Setup stage 1 completed successfully page, make sure the Create initial image of subordinate DIB node check box is selected (it is recommended to do this immediately). However, you can go back to this step at any moment during the data synchronization setup procedure.

    • After selecting the check box, click Done. You will be asked to create the initial DIB image. To do this, click Create initial image.
    • By default, the settings file required for DIB node creation will be created in a folder selected for exchange purposes, On this computer or in local network.
    • However, you can use radio buttons to select creating On 1C:Enterprise server. In this case, you will need to enter Infobase parameters:
    • Enter 1C:Enterprise server cluster.
    • Type in Name of infobase in cluster.
    • Select parameters in the Secure connection field.
    • Select DBMS type.
    • Specify Database server.
    • Type in Database name and
    • Fill in the Database user field and enter User password.
    • Specify Date shift.
    • Select Interface language.
    • Select the Set scheduled job lock check box if necessary.
    • Click Create initial image.
    • Save the file containing the initial image of subordinate DIB node to a new folder, which will be further used as a working folder for the new infobase.
    • Create a new infobase via the Add infobase to list command. Specify path to the initial image file. The selected infobase will be added to the infobase list.
    • Once you sign in to the application, the Wizard will automatically proceed from the Continue setting up data synchronization with master node page. Click Next to proceed with the setup and specify a master node connection method.

    Creating DIB nodes (external files)

    • If any of your files are stored in volumes, during initial image creation you will need to specify path to the .ZIP archive file containing all files stored in volumes. When starting the application with a local branch infobase for the first time, you will need to execute the Store initial image files command and specify the path to the archive file.

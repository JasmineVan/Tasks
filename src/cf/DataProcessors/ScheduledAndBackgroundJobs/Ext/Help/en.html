This form is used to administrate scheduled and background jobs.

A number of actions in the application can be executed automatically in background mode, on a specific schedule. Such actions are scheduled jobs. For example, it is daily currency rates import, regular report mailing, and deferred document posting. Scheduled jobs allow you to perform routine tasks in the application, and at the same time reduce the human factor (for example, you can forget to import currency rates).

To open this list, in section Administration - Service - Scheduled operations, click Scheduled and background jobs.

The list has the following features:

  • Scheduled jobs.
  • Background jobs.
  • Scheduled job execution monitor.

Scheduled jobs

In the list you can enable or disable scheduled job and change schedule.

The list includes the following columns:

  • If check box in the column is selected, scheduled job is Enabled and is executed according to the specified schedule.
  • Description. This column displays full description of scheduled job.
  • State. This column displays scheduled job execution details.
  • End date. This column displays date and time when the scheduled job is completed.

If the State and End date columns are <not defined> and the check box is selected, check scheduled job schedule. Perhaps the schedule is incorrect, so the scheduled job cannot be executed.

Refreshing lists

  • To update information in scheduled job list and on the Background jobs tab, click Update. Since the lists are linked, the list will be filtered according to the specified settings.

Schedule settings

  • For each scheduled job, a schedule can be set, according to which the scheduled job will be automatically started. The application supports one-time and periodic schedules.
  • Click Configure schedule.

Scheduled job execution

  • To start scheduled job execution immediately, click Execute now.
  • In file mode procedure will be executed in the current session.
  • In client server mode procedure will be executed in the background job on server. When a procedure is executed in a background job on server, the last execution date, which is used for execution by a schedule, is not changed.
  • Simultaneous execution of two versions of the same scheduled job is not allowed. If the same scheduled job will be started at the same time, the application will display an error message that the scheduled job cannot be started again.

Entering new scheduled job

  • You can also set two separate schedules for one scheduled job and add a new scheduled task following one of the predefined templates of the application.
  • Click Add, select a template from the list, and click OK. Fill in the required fields.

Enabling scheduled job

  • You can Enable or Disable scheduled job using the command of the context menu by the right click.

Background jobs

According to the specified schedule, a background job is created to execute the scheduled job, which executes scheduled actions. For example, if you set the daily schedule to 09:15 AM, a new background job session starts every day at 9:15 AM. After scheduled operation is completed, the background job is automatically completed.

The list includes the following columns:

  • State. This column displays execution of a job.
  • Description. This column displays background job description.
  • Start. This column displays date and time when the background job is started.
  • Finish. This column displays date and time when the background job is completed.
  • Scheduled job. This column displays scheduled job description.

To control scheduled job execution, use background jobs filter.

Filter Details
By state
  • Allows to filter by a state:
    • Completed.
    • Completed with errors.
    • Canceled.
    • Executing.
  • Select the check boxes for necessary filters.
By period
  • Allows to filter by a period:
    • Custom. You can specify a custom period.
    • All time. No filter by period will be applied.
    • Last night. Standard period of 12 hours starting at 9:00 PM till 09:00 AM.
    • Yesterday. Standard period of 24 hours starting at 12:00 AM till 11:59 PM of yesterday.
    • Today. Standard period of 24 hours starting at 12:00 AM till 11:59 PM of today.
By scheduled job
  • Allows to filter by a scheduled job:
  • Select the Filter by scheduled job check box.
  • Select scheduled job name from the list.
  • The scheduled job with the name "<not defined>", is used to filter background jobs,
    started for execution of other (not scheduled) jobs.

Warning! To filter background jobs according to the selected settings, click Update.

Viewing information about background job

  • To view background job properties, double click the line. You can also click Open.

Scheduled job execution monitor

  • The monitor view can be simple or detailed.
  • Simple view includes analysis of the State and End date columns on the Scheduled jobs tab. It is also includes analysis of the Messages to user and error details field in the scheduled job edit window.
  • Detailed view includes analysis of background jobs on the Background jobs tab that were created and executed to execute scheduled job.
  • Detailed view displays background jobs filtered by a specific scheduled job and analysis of background job attributes. Note, that scheduled job execution details are obtained by selection of a part of details of the last background job execution.
  • The filter to background jobs is applied only after clicking the Update command, so after changing filter settings, execute this command.
  • For a detailed analysis of the scheduled jobs execution in client server mode, see the Duration of scheduled jobs report from the Event log analysis report group. To open it, execute the matching command in the Administration - Service - Administrator reports section.

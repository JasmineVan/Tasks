Description of background job properties.

  • Description. This field displays copy of the field Description of scheduled job. If the Execute now command on the command bar of the scheduled job is executed, the description of the background job will have following value "Manual start: <Scheduled job presentation>".
  • State. The following values are possible: "Job is completed", "Job is completed with errors", "Job is canceled by the user", "Job is executing".
  • Start date. This column displays date and time when the background job is started.
  • End date. This column displays date and time when the background job is completed. If the background job is still executing the date and time are: <>).
  • Execution location. Name of a computer inside a network without domain on which the background job was executed. It either is a server name in client/server mode or a user computer name in file mode.
  • ID. This field is to be used only by an experienced administrator able to use 1C:Enterprise language. It helps to investigate abnormal problems of background jobs operation when they occur.
  • Key. The value of the Key field that was specified in scheduled job (at the moment when the background job was created) or another value that was specified explicitly if the background job was created from 1C:Enterprise script language, not by server scheduler.
  • Metadata method name. The name of the procedure in the configuration that is executing or was executing in the background job. It usually is the same value as value of the Method name field of the matching scheduled job.
  • Description. One of the field values is selected (in order of searching for filled fields):
    • Description.
    • Synonym.
    • Name.
  • ID. The ID of scheduled job matching the background job.
  • Message. This field contains warning texts and error details if errors occurred during the background job execution.

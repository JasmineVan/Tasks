Description of scheduled job's properties.

The need to change settings of scheduled jobs is seldom. You can deselect the Use check box to disable a scheduled job.

You can also set two separate schedules for one scheduled job and add a new scheduled task following one of the predefined templates of the application.

Scheduled job properties

  • Use flag allows to enable and disable a scheduled task.
    This option is useful when you need to disable a scheduled task for a short time without changing the schedule.
  • You can set the scheduled job schedule via the Schedule link.
  • Description property field provides description for a scheduled job. This property field is usually blank, and the synonym that is set in a scheduled job template in Designer mode is used instead of it. If the synonym of a template is not set, then its name is used.
  • In case of errors, repeat __ times is used to indicate the number of attempts to run the scheduled job in case errors occur.
    • after __ seconds indicates a time gap (in seconds) between the attempts to run the scheduled job in case errors occur.

  • Predefined property field has either a Yes or a No value. Non-predefined (manually set) scheduled jobs are rarely used.
  • ID property field is to be used only by an experienced administrator able to use 1C:Enterprise language. It helps to investigate abnormal problems of scheduled jobs operation when they occur.
  • Key value property field is to be used only by an experienced administrator able to use 1C:Enterprise language. It is required for fine-tuning execution of scheduled jobs and investigating problems in their execution when they occur. If key value is set, before running a background job, the application checks if there is no other active one with the same scheduled job template as the current background job.
  • Username is a name of an infobase user that will be used for the execution of a scheduled job. Accessible from the Users list.
  • Name is a template name for a scheduled job set in the Designer mode.
  • Synonym is a synonym of a scheduled job template set in the Designer mode.
  • Method name is a name of a procedure in a configuration that will be run in a scheduled job created following the scheduled job schedule.
  • Message field is used for warning messages and error details if they occur during the latest execution of a background job created within the scheduled job. The same content is available, if you open the latest background job of this scheduled job.

This form is designed to administer accumulation and accounting registers, and aggregates of turnover accumulation registers.

Registers are designed to store and process information about the activities of the enterprise. Registers usually store information about changes in the objects state of the object domain (documents, catalogs) or other information not recorded directly in these objects. For example, registers can store information about currency rates.

Aggregates are a mechanism available in turnover accumulation registers. You can create multiple aggregates for any register. Each aggregate is a dedicated storage that contains aggregated register data broken down by certain parameters. This streamlines report generation. Aggregators can significantly reduce the time of report generation. This is important for large infobases.

To open it execute the matching command in the Administration - Service - Scheduled operations section.

Totals and aggregates management includes the procedures related to the totals of accumulation and accounting registers and aggregates of turnover accumulation registers. Totals management is divided into two modes:

  • Frequently used features. This mode provides simple tools to perform the most commonly used actions with register totals. This mode allows you to execute the most common actions in "one-click" that occur during maintenance of the application.
  • Full functionality. This mode provides full access to the management of the totals and aggregates of the application. This mode allows you to execute all operations with one register as well as with an arbitrary list of registers.

To switch modes, use the reference in the lower right corner of the window. The application stores window open mode.

Some operations with registers and aggregates can be executed according to the specified schedule, using scheduled jobs.

Frequently used features

Totals operations

  • Set calculated totals period. This operation sets calculated totals period for all accumulation and accounting registers with enabled totals, to accelerate the application operations. For accumulation registers it sets at the end of the previous month, for accounting registers it sets at the end of the current month. You can use the command at the beginning of each month to improve register performance.
  • Enable totals use. This operation enables the use of totals for all registers that have this mode disabled (for example, as a result of the abnormal termination of a procedure that disables the use of totals).

Aggregates operations

  • Rebuild and fill in. This operation rebuilds aggregate usage and fills in aggregates for all turnover accumulation registers which have aggregate mode and use enabled. Operation can take a long time. This operation, as a rule, must be executed periodically, for this you can use the scheduled job.
  • Receive optimal aggregates. This operation receives the list of optimal aggregates for all turnover accumulation registers which have aggregates set in Designer. The directory will be requested where the XML files with the lists of optimal aggregates will be saved. Each file will be named with the name of matching register. For the Sales register, file will be named Sales.xml. Files are used to import and create optimal aggregates in Designer. When importing optimal aggregates, you can need to change list of optimal aggregates, and for this it is necessary to disable configuration support. The operation is not regular. This operation is the most resource-intensive and time-consuming, so it is recommended to execute it in cases when other users do not use the application.
    • The calculation of optimal aggregates can be required in the following cases:
      • Significant reduction in the report generation speed in the current list of aggregates.
      • Significant change in data stored in the application.
      • Significant change in the reports that users need.

Full functionality

This mode allows you to get access to all administrative tools for register totals (Totals tab) and aggregates (Aggregates tab). In full functionality mode, multiple selection mode is available. Any action is applied to all selected objects, or to the one over which you hover the mouse pointer.

The list shows the current state of the application registers.

Totals

The Totals tab contains a list of accumulation and accounting registers.

The list includes only those registers that have the Totals management right for the current user.

The list shows the current state of the application registers.

The list includes the following columns:

  • Totals. If the check box is selected in this column, totals are calculated for the register, and if the check box is not selected, the register totals are not calculated.
  • Register and use mode. This column displays description of the register and use mode of the register in the tooltip: Just total register (if the register has balance), Totals or Aggregates (if the register has only turnovers).
  • Period. Date until which the register totals are calculated (for accumulation registers it sets at the end of the previous month, for accounting registers it sets at the end of the current month).
  • Current. If the check box is selected, not only general totals for the specified date are calculated for the register, but also real-time totals for the date of the last record. If the check box is not selected, real time totals are not calculated for the register. It makes sense to use this mode for registers whose records are analyzed in real-time accounting before the documents posting.
  • Split. If the check box is selected, totals separation mechanism is used for register, which provides improved parallel writing to the register.

Modes that cannot be changed with the current application state are marked in gray. You can switch modes on the Aggregates tab.

You can execute the following operations on this tab:

  • To update information about total state, click Update.
  • To enable or disable totals use, click Totals.
  • To enable or disable current totals use, click Current totals.
  • To enable or disable totals separation, click Totals separation.
  • To Set selected period of calculated totals, click Set calculated totals period.
  • To recalculate the totals, recalculate the current totals, and totals for the selected period, click Recalculate.

Aggregates

  • The tab in the upper list contains a list of turnover accumulation registers, for which aggregates are specified in Designer, and the lower list contains the list of these aggregates. The list of aggregates contains the flag that shows whether aggregate is used and statistical information about it.
  • In the Register column, in addition to register description, information about register use mode is displayed: Total data is used or Aggregate data is used.
  • The list includes only those turnover accumulation registers, for which the current user has the Totals management right and aggregates are specified in Designer.

You can execute the following operations on this tab:

  • Set register Mode: totals or aggregates.
  • Enable or disable aggregates Use.
  • To rebuild aggregate usage, click Rebuild.
    • Specify rebuild parameters.
  • To fill in aggregates, click Update.
  • To Clear aggregates, click Clear aggregates.
  • To calculate list of optimal aggregates, use the Optimal command. Specify parameter of optimal aggregate calculation. After that, the directory will be requested where the XML files with the lists of optimal aggregates will be saved. Each file will be named with the ID of matching register. For the Sales register, file will be named Sales.xml. After the operation is completed, in the registers list the registers for which it is necessary to update the list of aggregates to the calculated optimal aggregates will be marked in bold. The list must be updated in Designer.
    • Specify parameter of optimal aggregate calculation.
  • To Update information about aggregates, click or F5.

In the lower part of the window, a list of register aggregates is displayed, on which you hover the mouse pointer in the upper list.

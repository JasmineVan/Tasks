Two complementary infobase update methods are available for your convenience:

  • using exclusive processors:
    • all users are locked out.
    • this method is used to update data that must be fully processed prior to the users logging back on.
      • the less data volume, the shorter downtime.
  • using deferred processors:
    • Users can sign in to the application.
    • this method is used to update the infobase "data archive" (i.e. data not actively accessed by the users, or data that can be accessed by the users while the update is running).

Infobase update

  • The first application startup after the update must be made by the administrator in Designer or with the wizard. At first run, the application finalizes the update. Depending on the update scope and computer performance, it might take time.
  • We recommend that you terminate all user sessions before running the application for the first time after update. This is necessary for correct application update.
  • If any user sessions in the application are active, you can log these users out in the file mode and temporarily lock the authorization. In the client/server version, the start is aborted and the "active user sessions" message is displayed. Follow the link to view the active users list and ask them to end their sessions or to proceed with an update by clicking .
  • The application will prepare active user sessions to being ended. All active users will be asked to sign out. The application will be locked and the update will be resumed.
  • However, the application will not shut down; it will continue running and display the on empty desktops. The message displays the new infobase version number. The indicator displays infobase update progress - both visually and as a percentage. To view detailed information on the infobase update progress, click in the event log.
  • If an obsolete platform version is used, the application will display a message that requires to update the platform version and lock the desktop.
  • The application restarts automatically once the update is finished. The application desktop will open, but the application will be locked. To unlock it and start working with the application, confirm the .
  • If the update is interrupted by anything, the application displays the asking to contact technical support. To view such error messages, click . Users will not able to work with the application until the issues are resolved.
  • In case of successful update, the application is unlocked and the window emerges for all users .

Features and advantages of deferred infobase updates

  • In Client/server mode, application update may require additional data processing that should be executed in background to reduce downtime. Users can start working with the application before the processing is completed. White the update procedure is running, certain application functionality may be temporarily unavailable.
  • In this mode, the documents that have not been processed yet will become temporarily unavailable. If you try to open any of such documents, the program will display the message warning that actions with documents are blocked for the time the application is being updated. Some reports may also become unavailable during the update. When a user attempts to run a report, they get a warning that the report might contain inaccurate data because the update is in progress.

This form is used to permanently delete objects marked for deletion.

While working with applications, users may need to mark certain objects for deletion (for example, objects created by mistake). Objects marked for deletion can be permanently deleted by a user with administrative rights.

To delete a marked object, execute the Deletion of marked objects command located in the Administration - Service section.

During deletion procedure, reference integrity control is performed. This means that if a particular applied object (document, catalog item, etc.) is referenced by other objects, the deletion of the object will not be performed. Keep in mind that deletion of the marked objects is an irreversible operation.

When deleting the marked objects, two methods are available (selectable by radio button):

  • Delete all marked objects automatically . In this mode, all objects marked for deletion will be deleted permanently.
  • Delete objects selectively. In this mode, the user selects which objects to delete

Automatically delete all selected objects

  • Click Delete.
  • In this mode, all objects marked for deletion will be identified and deleted. Once the procedure is complete, a notification containing number of deleted objects will be displayed. User intervention is only required when a marked object is referenced by other objects and cannot be deleted.

Delete objects selectively

  • Click Next.
  • In this mode, a list of objects marked for deletion will be generated.
    • Objects in this list are grouped by type. All check boxes are selected by default.
    • If you do not want to delete some of the marked objects in the list, clear the corresponding check boxes.
    • To open the object and decide if it should be deleted, double click the object or use the Change button.
  • To delete the selected objects permanently, click Delete.
  • Once the procedure is complete, a notification containing number of deleted objects will be displayed.
  • If any of the marked objects were referenced by other objects, the application will generate a warning and display a list of marked objects that were not deleted due to references, as well as a list of objects referencing them.

How to proceed when an object cannot be deleted

The list of objects that could not be deleted consists of two sections: object list on the left, and Usage instances on the right.

  • To delete these objects permanently, you need to:
    • Double click it in the list or using the More - Change command to open the object that prevents deletion, for example, to manually delete all its references to the object being deleted.
    • replace the reference that prevents deletion. Click Replace everywhere where found to proceed to replacing items. To replace a reference, select a replacement item from the list.
    • Mark to delete all objects associated with the item to be deleted (Usage instances on the right),
    • Clear deletion mark from the marked object.
  • Then click Retry deletion to retry object deletion.
  • Click Restart to select a different deletion method.
  • Click Close to cancel the deletion procedure and go back to the application.

Accelerate deletion of marked objects

  • Deletion of marked objects can take a long time. While deleting the user experience may suffer.
  • We recommend that you delete marked objects in the periods of the least user activity.
  • You can Block all operations in the application and speed up the deletion selecting the corresponding check box.

Configuring a schedule of automatic deletion.

  • Select the check box to Automatically delete selected objects on schedule.
  • After that the Configure schedule link becomes available. After the schedule is configured, it is displayed below the check box.

Automatic deletion of marked object of the application n the Interned (in the SaaS mode)

  • The Internet application (in the SaaS mode) does not have setting of marked object deletion on schedule, the Configure schedule link is missing.
  • This is due to the fact that you cannot configure scheduled of scheduled jobs in the Internet application.
  • That is why the check box is called Automatically delete selected objects. Select the check box to run background deletion of selected objects.

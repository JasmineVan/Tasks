After the attributes change value is entered, you can customize the settings by using the Additional parameters command:

  • Change in transaction (the check box is selected by default) – the program allows processing the elements of the list in transaction selected to be changed. If any of the objects fails to be recorded while the check box is selected (for instance, due to it being edited by another user), changes in non of the objects will be saved.
  • Abort on error - it is recommended that you select this check box if changes of objects are performed outside the transaction; the check box is set by default. In this case, if it is impossible to record an object, the program displays a warning and offers either to continue changing the rest of the objects or to abort processing to establish problems. If the check box is not marked and the object cannot be recorded, the problem skips it and changes other objects.

  • Process subsidiary elements - this check box is not selected by default; it allows to alter subsidiary elements of selected groups and parent objects without having to select all check boxes of all elements of groups and subsidiary objects. If the check box is not selected, the program, processes the group itself, not its subsidiary elements.

Additional parameters for an administrator

There is an additional parameter for an administrator, which you can manage by executing the Group attributes change command in the Administration - Support and service:

  • Special features - by default, in the Group objects change window, the list of available attributes does not include the ones which are not recommended to alter. To view them, click . Since editing service attributes can affect performance of the program, it is recommended to use these features with caution and only for maintenance. The following parameters are available:
    • Show service attributes - mark this check box to display the attributes that are not recommended to alter;
    • Developer mode - Execute record of objects in the Data.Exchange.Load = True mode;

    • Disconnect filters on selecting values - when selecting values, this parameter allows not to use filters written in properties of the ChoiceParameterLinks and ChoiceParameters attributes.

  • Process subsidiary elements parameter is unavailable in this mode.

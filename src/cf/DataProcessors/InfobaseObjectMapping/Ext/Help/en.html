This list is used to perform data mapping between two applications.

You can access the list via the Execute mapping command on the Data mapping page of the Interactive data synchronization wizard. Or double-click any row with data mapping problems.

The list contains two columns, each one dedicated to an infobase participating in data exchange. The list data is grouped by applied object (documents, lists). A status bar is displayed at the bottom of the list, showing how many items are mapped and how many are not.

The Output field is used to control which data is displayed in the list. The Unmapped data is displayed by default.

Object mapping

  • Click Map automatically (recommended) and select the fields for mapping via check boxes. Some of the fields are selected by default. Click Execute mapping to confirm. The application will perform mapping and then display the list of mapped data. Click Apply to confirm.
  • Now you can perform manual mapping for objects that failed automated mapping, or edit the automated mapping results. To create a mapping between two objects manually, select the objects and click Set mapping. To remove a mapping between two objects, select the objects and click Cancel mapping.
  • Click Write and close to confirm.

Setting up fields for mapping tables

  • Click Columns to add fields to the list columns. Select the additional fields via check boxes and click Apply to confirm.

Importing data from target application

  • To receive data from another application, click More actions - Import data from another application.

Object mapping procedure

  • We recommend that you consider reference links when performing data mapping and data import. This recommendation is crucial for fields used for object mapping.
  • For example, a configuration includes a counterparty contract catalog subordinate to the counterparty catalog. The counterparty contracts are mapped based on the owner catalog, this being the counterparty catalog. Therefore, to avoid mapping errors, you first need to map and import the counterparty catalog, followed by the counterparty contract catalog.
  • Otherwise, mapping table fields can contain dummy references like:
    <Object not found> (26:a0b9001b24e002fe11dfba347dd41412).
  • Dummy references point to the current infobase objects that are not yet imported from data exchange messages.

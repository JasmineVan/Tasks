Designed to configure infobase backup.

Sometimes data loss may occur, for example, due to power failures, computer failures, or incorrect actions in the application. To reduce risks of data loss, the application administrator must regularly back up infobase. Only in this case you can restore the infobase. The frequency of creating backups depends on the intensity of new data entry into the application. It is also recommended that you crate backups before any operations in the application that can change large amounts of data, for example, before bulk editing objects.

Data can be restored from a previously created backup file at any time. Please consider that the recovered infobase contains only data that was entered at the time of creating a backup. Data entered after the backup will be permanently lost. Thus, it is important to create a backup schedule to minimize data loss.

The use of backup in 1C:Enterprise mode is available only for file mode. It is recommended that you use DBMS tools in client/server mode. For more information, see 1C: Enterprise 8.2. Client/server mode. Administrator Guide.

You can open it by clicking a corresponding link in the warning that is displayed on launching the application if backup is not configured. This warning offers you to configure infobase backup. If you ignore the warning about backup setting, it will be displayed every hour.

You can also click Backup setting in the Administration - Service - Backup and restoring section. Under this command, you can see information about backup status in the application.

Restrictions

  • Warning! Backup is not supported if the Administrator user on whose behalf the backup is running has operating system authentication.

Backup setting (file mode)

  • The application allows you to create backups in the 1C:Enterprise mode. Backup can be performed automatically or manually. To perform backup automatically, configure it.

  • Select the Run automatic backup check box.

  • After that, you can select the following options using the radio button:
    • Regularly on schedule. Backup is performed automatically. Follow the link to set up backup schedule. The link shows the entered schedule. Unavailable in basic versions.
      • When creating a schedule, make sure that the application operates at the time the backup starts. Otherwise backup will not be performed.
      • Warning! Some standard settings in the backup schedule are not supported: Repeat after pause, Terminate at, Stop in. Select other settings in order to replace them.
    • On exiting. On exiting the application, administrator is asked to perform a backup. To start backup when exiting the application, confirm you choice by clicking Exit. The Backup execute check box in the Exit application window is selected by default. Clear the check box if you do not want to be prompted to back up data on exiting the application.
  • Select the Directory for backup saving. To do this, click Select, specify the path to the folder where backups are placed to, for example, E:\Backup. The field is required. It is recommended that you back up to another computer or an external storage media.
  • Using the radio button, select a period during which the application will Store backups, or the number of stored backups:
    • All. You can select this option if storage capacity of your hard drive with backup directory is enough to store backup files.
    • Over the past 6 months. (by default). You can select the period of backups storage.
    • Last 10 pcs. (by default). You can restrict the number of stored backups.
  • Click Finish to confirm your choice and go back to the application. The application will switch to automatic backup mode.

Performing backup by third-party software

  • You can decline automatic backup setting if backup is performed by a third-party software, for example, by DBMS in client/server mode, or third-party backup software.
  • To do this, clear the Run automatic backup check box.

Backing up on file storing in volumes

  • If files are stored in volumes on a hard drive, back up these files by using other tools.

Backup errors

If the backup failed, and there is the following entry in the temporary log file: "Exception in createConnection: Error, Automation server can't create object", then the 32-bit comcntr.dll component is most likely used on a PC running a 64-bit operating system. To fix this problem, take 64-bit comcntr.dll component from the 1C Enterprise x64 Server distribution, and register it by running from the command line:

regsvr32 <64-bit comcntr.dll location>\comcntr.dll

Important: To sign in to the infobase if backup is not performed, start 1C:Enterprise with parameters: ENTERPRISE /F <path to the infobase> /C AllowUserAuthorization /UC Backup.

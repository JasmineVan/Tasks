Export and import EnterpriseData data

The data processor is designed for data export and import using the EnterpriseData universal format. It is used in infobases supporting data synchronization via the universal format.

Operations of the configuration built-in data processor

The configuration built-in data processor operates only in data import mode.

To import data, specify a path to a file with data to be imported and click Execute.

To export data and to use additional features, use the external version of this data processor, which is included in the package.

Simplified run mode

To export data:

  1. Specify a Path to export file that will be used as a destination for data export
  2. Specify a Format version to be used for data export
  3. Set up the content of data to be exported:
    1. Select metadata objects
    2. If necessary, specify a filter by period for documents
    3. If necessary, set up arbitrary filters for catalogs and documents
  4. You can save filter settings to a file and use it in next sessions (to do this, use the Save settings and Restore settings buttons).

 

To import data, specify a path to a file with data to be imported

Developer mode

To enable the mode, click the More button. The mode has the following features:

  1. Export and import data from text. In this case, instead of a file, a text field in a form is used as a data source and data destination. This mode can be helpful if the amount of data is small and it is convenient to move the export result to the import data processor via the clipboard.
  2. Using external exchange modules with the help of the universal format. To do this, specify a path to an external data processor which module contains the external module of the exchange manager.
  3. Export of data registered at the exchange plan mode. To do this, specify the Exchange node during the export. In this case, exported data contents settings and filter settings are unavailable.

This panel is used for setting up parameters for import, storage, and processing of files. The following commands are available:

  • Store files in volumes on disk - used to select between storing files in the infobase or in volumes on a disk. Storing files on a disk presupposes that they will not be stored in the application.
    • In the check box is selected, the File-keeping volumes command becomes available. It allows to set up access to file-keeping volumes (folders) on a file server.
  • Prohibit importing files with extensions - select this check box to prohibit uploading to the application the files that contain certain extensions and enter the names of extensions to prohibit to the input filed separating them by spaces.
  • Maximal file size - enter the limiting maximal size of files imported to the application. The default value is 50 MB. The files that exceed this size will not be imported into the application. To make the size of files that can be imported unlimited, set the maximal size to 0.
  • ODF flies extensions is used to set extensions for files of open document format, whose text will be extracted to build full-text search index. Input extensions separating them with a space. By default, the list contains basic file extensions. You can create or edit the list of extensions of files of open access format only if you are using Microsoft operation systems and if an OpenOffice application does not have an IFilter extension.
  • Extensions of text files is used to indicate extensions of files containing text data. Input extensions separating them with a space.
  • File clearing settings - execute this command to set up clearing of unwanted files.
  • Sync files with cloud services - select this check box to synchronize files with Yandex.Disk, Box, or other cloud services and facilitate editing them.
  • Sync settings - follow this link to set up synchronization of files with cloud services.

Advantages of keeping files in volumes on disk

  • It takes unnecessary load off the infobase. This kind of files can be very bulky, and their volume can significantly exceed the volume of all other infobase data. In such case, operation of the application significantly slows down;
  • The size of a volume is only limited by the size of a hard disc;
  • You can check files for viruses;
  • It is a simple way of having files backup.

Advantages of keeping files in the infobase

  • All data is stored in one place - the infobase;
  • Standard 1C:Enterprise or DBMS means are enough for backup.

However, keeping files in the infobase is useful only for bases of a small number of files (size).

Rules for storing files in volumes

  • If you select or clear the Store files in volumes on the hard disk check box, you will no be able to cancel your changes. If you disable volume file storage, new files will be added to the infobase while the old ones will remain in volumes. A volume folder will be used until the infobase stores at least one file. Files that were previously saved to volumes open from these volumes.
  • If you save a file in a volume, and then delete the volume by external means, the file will not be found even if the Store files in volumes on the hard disk check box is already disabled. Selecting or clearing the check box will not bring the missing files back.
  • If the application cannot find the files stored in volumes, you should look for error messages in the event log, see the path to the volume folder, check the location of this folder, make sure it contains the file you need, and restore the folder if necessary. To be able to restore folders, make sure you perform regular back-ups of volume folders by external means, for example, by operating system means.

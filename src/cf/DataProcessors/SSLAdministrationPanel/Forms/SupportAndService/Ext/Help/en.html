This panel is used to control application status, create backup and recovery, update an application version, and optimize operability.

  • Event log
    		•  Used to view events and actions of scheduled jobs or application users.
  • Active users
    		•  Lists users that are working with the application at the moment.
  • Application lock
    		•  Used to lock and unlock operations of users in the application. When lock is on, all sessions of active users (including the current one) are terminated.
  • Deletion of marked objects
    		•  Used to delete objects of the application that were marked for it.

    Scheduled operations

    This subsection offers commands for managing scheduled jobs:

  • Scheduled and background jobs
    		•  Setting up, execution, and tracking status of scheduled and background jobs. Scheduled jobs carry out operations on schedule.
  • Unlock operations with external resources
    		•  Use this button to unlock scheduled jobs responsible for work with scheduled resources. The lock is set automatically when infobase changes location due to it being copied or restored using a backup copy.
  • Totals and aggregates management
    		•  Used for managing totals of accumulation and accounting registers and aggregates of turnover registers. Setting up totals and aggregates correctly can increase productivity of operations in the application.

    Backup and recovery

    To minimize risk of data loss, run backup copying regularly. Frequency of creation of backup copies depends on the amount of new data being stored.

    Backing up Creating application backup copy: right now, in some time, or on finishing work with the application.
  • Backup setup
    		•  Used to set up automatic backup on schedule or switch off backup control if it is performed by external means.
  • Restore from backup
    		•  Used to restore the application from a backup copy.

    Data adjustment

    The application offers various means of keeping lists in order: editing data filtered by certain features; automatically deleting repeated elements and correcting links to deleted elements.

  • Bulk attribute editing
    		•  Change attributes and tabular sections in the selected items.
  • Duplicate object detection
    		•  Search similar items by the specified comparison conditions.
  • Accounting check rules
    		•  Contains rules for checking infobase data relevance.

    Update application

    This subsection is used to install application updates and patches. You can also view unformation on previous updates here.

  • Installation of updates and patches
    		•  Used to install updates and patches.
  • Installed patches
    		•  Used to view the list of patches installed in the application.
  • Change log
    		•  In this subsection you can find notes on changes introduced in a new application version and recommendations on application update.
  • Update results and additional data processing
    		•  Information on progress of the application version update, deferred execution of additional data processing procedures.
  • Detail update process in event log
    		•  When this check box is selected, data on the update handlers being executed and their execution time is stored in the event log.

    Performance monitor

    The application offers built-in means of collecting and analyzing data on its productivity following APDEX. This subsection allows to set up the following parameters:

  • Performance monitor
    		•  Select this check box to measure integral productivity using the APDEX method.
  • Key operation profiles
    		•  Used to set up profiles of key operations.
  • Performance assessment settings
    		•  Customize performance assessment parameters.
  • Export performance measurement data
    		•  Used to export performance measurements for an arbitrary period.
  • Import performance measurements
    		•  Used to import performance measurements for an arbitrary period.

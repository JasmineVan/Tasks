This panel is used to manage users and set up access rights. The following commands are available:

Users

This subsection contains grouped commands for managing users of the application and setting up access to it.

  • Users - allows to manage users that are working with the application.
  • Access management - allows to go to settings controlling passwords change and the level of passwords complexity. You can set periods passwords are valid without changing and limit operations of inactive users.
  • User groups - select this check box to be able to manage groups of internal and external users. One user can be a member of several groups. Access rights that are set up for a group are applied to all its participants. Groups are useful when the application has many users.

External users

This subsection contains grouped commands for managing external users of the application and managing their access to it.

  • Allow access to external users - select this check box to manage access of external users to the application and use them in it.
  • External users lists users that are working with the application "from the outside". Usually external users are partners, respondents of surveys, clients of internet shops, etc. Management of external users is similar to management of the internal ones.

  • Access management is used for managing complexity of passwords, setting the terms for changing them, and limiting operations of inactive external users.

Access groups

  • Access groups is used for setting up access rights and limitations for users and user groups. An access group defines a set of actions with the application data the members of this group are allowed to perform.
  • Limit access on the level of records - select this check box to be able to flexibly set up access of users to catalogs, documents, and other parts of the application.
  • Access group profiles is used to set up templates for user access groups. As a rule, access group profiles contain several roles. Once a user is added to an access group, they are automatically assigned all roles listed in the access group profiles.

Personal user settings

  • Copy settings is used to apply one users's settings of program operation context to another. This can be useful when creating a new user.
  • Clear settings is used to clear settings of a user, a user group, or all users. This restores consistency of settings and in some cases helps to normalize operation of users' sessions.
  • User settings is used to view and manage settings of a user.

Period-end closing dates

  • Period-end closing dates - select this check box to restrict retrospective changes to data recorded before the end date of the period. This activates the following command:
  • Set up - use this command to restrict editing application data recorded before a stated date.
  • See also Data import restriction date on the Administration panel - Data sync.

Personal data protection

  • Register events of access to personal data - used to set registration of access to personal data (Russian Federation Federal Law N152-ФЗ from July 7, 2006, "On protection of personal data");
  • Personal data protection is used to view addresses to personal data of the selected objects of the application.
  • Hide personal data in - use this check box to hide personal data after a stated period of time (for example, after seven days). The reason for hiding data can be that the subject's consent is no longer valid or is missing.

In this panel you can synchronize data with other applications and organize collaboration in the distributed infobase.

  • Data synchronization. Select this check box to activate the Data synchronization application section and commands:
    • Data synchronization settings. Proceed to setting and synchronizing data with other application (for example, with 1C:Accounting, version 3.0), as well as to organizing collaboration in a distributed infobase.
    • Import closing dates. Select this check box to close import of previous period data from other applications.
      • See also: Period-end closing dates in the Common settings subsection.
      • After it the following command becomes available:
      • Customize. Proceed to closing data of previous periods to import.
    • Warnings on data synchronization. View warnings, appeared during data synchronization (unpassed documents, unfilled attributes, conflicts and data not received by closing date).
    • Import Enterprisedata data. Import data in the Enterprisedata format without needing to set regular data synchronization. Infobases should support data synchronization through the universal Enterprisedata format. Using this command you can only import data.
    • This infobase prefix is used to define the program in which list items and documents were entered. Infobase prefix is added to list item codes and document numbers automatically.

Temporary file directory of 1C:Enterprise server cluster

  • Path to a single network directory for common temporary files of 1C:Enterprise servers working in a single cluster:
    • Managed by Windows OS. Specify the path to a united temporary files directory of all servers in the cluster, which will be used for servers managed by Windows OS.
    • Managed by Linux OS. Specify the path to a united temporary files directory of all servers in the cluster, which will be used for servers managed by Linux OS.
  • Temporary file directories can be left blank when only one server works in the 1C:Enterprise cluster, or the application operates in file mode.
  • If all servers in the cluster are running under the operating systems of the same family (Windows / Linux), then the second field is optional.
  • Directory should be available for reading and recording for all servers of the cluster. Including all users of operating systems, on behalf of which 1C:Enterprise servers are started.
  • UNC directory path should be the same for all servers of the same family. For this directory should be connected to servers under the same name.
  • In the Common temporary files directory in Linux field you can only specify the local path, for example, /home/path.
  • Before you can use UNC path, you need to mount the folder. Sign in as administrator (root) and execute the following commands:
    • mkdir folder
      write a directory name instead of folder. for example, make mkdir shared in the /home directory. This command will create the shared directory with the /home/shared full path.
    • mount -t cifs //server/share folder -o //domain/user, password=pwd
      , where //server/share is the UNC path to other computer, folder is the directory path on Linux computer, for example \home\shared, //domain/user is a user/domain name, if needed, and pwd is a user password (if needed).
    • If user name and password are not required, simply execute
      mount -t cifs //server/share folder
    • Now you can specify folder (example: /home/shared) as the path to the directory.

Use this window to work with files attached to an application object.

To open this form, follow the Files link in the record of an object.

The list includes the following columns:

  • Indicates that a file was signed with a digital signature or was encrypted.
  • Description. Full name of the file. An icon next to the name specifies file type. The column can also display detailed information about the file if such information exists.
  • Author. User who added the file.
  • Change date. Date of the latest file data change (local time).

Click More - Change form to display the following information:

  • Date created. Date and time the file was entered to the application.
  • Edited by. User currently editing the file; and so on.

Adding new files

  • Select Add. Select a method for file entering from the drop-down menu:
    • From disk.
      • specify a path to the file on your computer.
    • From template.
      • select a template from the list.
    • From scanner.
      • a scanning dialog box opens. Follow the instructions.
  • After you add a file, you will see an updated list of files.
  • The attached files are stored in dedicated folders. A separate folder is created for each applied object.
  • In some application lists, items with attached files are marked with icon .

Importing files

  • Click More actions - Import files and specify a file path on the computer.

Viewing images in the list

  • Select the check box by clicking More - Preview to view image files in a special window right in the list.

Viewing file cards

  • Click More - Open card or press F2 to open the Attached file window and see the properties of the file selected in the list. You can also select this command by right-clicking the file.

Editing attached files

  • In thin client, spreadsheet document files (.mxl format) are edited in embedded spreadsheet document editor. In web client, you need to download an application for editing spreadsheet documents. For more information about editing spreadsheet documents in web client, see this page.
  • To edit files in other formats, you need to have appropriate applications installed on your computer. For example, to edit files in Microsoft Word format, you need to install Microsoft Word.
  • The Open command opens an attached file in an external application in editing mode (if the file was not locked) or in viewing mode (if the file is locked for editing by another user).
  • The Edit command opens the attached file in editing mode and locks the file. Locked files are unavailable for editing to other users. In the list, they are marked in green.
  • After editing is finished, save the file, and then close the external application. Click Save and unlock file in infobase to release the file, save it in the application and unlock it for editing.
  • To unlock the file for editing without saving changes in the application, click More actions - Additional - Cancel editing. This command is useful to discard file editing results, for example, if some unwanted changes or mistakes are made in the file.

Printing files

Click Print from the dropdown menu and select the required action, for example, Directly to printer. There are other actions available.

Sending files by email

  • Click Send. The application generates an email. Fill in the required fields. The attached file is automatically attached to this email.

Save files

  • To save a file to the specified directory on the hard drive or a network drive, click More actions - Additional - Save as.
  • You can also use this command by right-clicking the file.
  • To Open file directory in the Windows Explorer, right-click the file or click More actions - Additional, and then select the corresponding command.

Updating files

  • To update an attached file from the original file stored on the hard drive (the source of the attached file), click More actions - Additional - Update from file on hard drive.
  • You can also use this command by right-clicking the file.

Comparing files

  • In the list, select files you want to compare.
  • Click More - Compare files. When you try to compare files for the first time, the application will prompt you to select an office application that will be used for comparison. After that, the files to compare open in the selected office application. The application for comparing files must be installed on your computer.

Digital signature and encryption

The Digital signature and encryption drop-down menu contains a set of commands for digital signature management:

  • Sign to create a digital signature for the file. Before creating the digital signature, you will be offered to select a digital signature certificate from the list of available certificates. You can also select a certificate upon customizing personal application settings. Digitally signed files are not available for editing. In lists, they are marked with icon .
  • Save with digital signature to save a digital signature to a file on the hard drive. For example, this command is useful if you need to deliver the signed file to another user.
  • Add digital signature from file to add a digital signature to a file. For example, this command is useful if the digital signature file was received later than the signed file.
  • Encrypt to encrypt a file. Unlike file signing procedure, you can use multiple certificates for file encryption, for example, if you need to encrypt a file for several users. In lists, encrypted files are marked with icon .
  • Decrypt to decrypt a file. To decrypt a file, you need to enter the private key password.

Printing spreadsheet documents with digital signature stamps

  • To print a spreadsheet document signed with a digital signature with digital signature visualization stamps, in the list of attached files, click Print - With digital signature stamp.
  • The spreadsheet document must be signed beforehand with one or several digital signatures according to the procedure prescribed in the document.

Service attached files (for administrator only)

  • While working with objects, the application can also attach to them service files, for example, files that are created automatically when sending info to bank and so on. They may be interesting to the developer, but the user must not see and edit them.
  • Select the More actions menu check box to Show service files. This check box is available only for users with the Full rights role.
  • Service attached files are for viewing only.

Encoding is a numerical schema where each text character is a certain numeric value. Usually different languages consist of different sets of characters. That is why there are a lot of encoding standards. Text you see on the screen is stored in the text file as a numeric value. Computer converts these numeric values by using an encoding standard and shows the text to a user.

You can choose encoding of a text file by selecting a relevant command when importing a file (importing a folder) from your hard drive to the file list. To import a file or a folder, right-click commands in the context menu of the list.

You can also Change encoding of an existing file in the built-in application text editor by selecting a relevant command in the More menu.

Selecting encoding of text files

  • Select an encoding from the list.
  • If there is no necessary encoding, clear the Show only the main encodings check box.
    • A list of all known encodings will be generated.
    • Select an encoding from the list of all encodings.
  • Click Select to save the setting.

If you open a file and see unknown characters instead of the text, change the encoding. The following encodings are typically used for files containing texts in Russian:

  • IBM866 (Cyrillic DOS)
  • KOI8-R (Cyrillic KOI8-R)
  • Windows-1251 (Cyrillic Windows)

General information

  • To prevent problems with encoding and decoding files, you can save files in a Unicode file format. Unicode contains sets of characters for the most languages used on computers.

Designed to create infobase backup.

It is recommended that you create backups before any operation in the application that can irreversibly change large amounts of data, for example, before bulk editing application object attributes. In this case, you should back up data manually.

It is accessible by clicking Create backup in the Administration - Service - Backup and restoring section.

Restrictions

  • Warning! Backup is not supported if the Administrator user on whose behalf the backup is running has operating system authentication.

Backing up

  • At top of window, you can see the last time the backup was performed.
  • By clicking Select in the Directory to save backup field (the field is required), specify a folder the application will save the infobase backup to. It is recommended that you back up to another computer or an external storage media.
  • If there are current sessions in the application, the application displays a warning about it. Click the link to View list of active sessions. User work in the application should be completed.
  • After setting the required parameters, click Save backup.

Backup procedure

  • The application attempts to back up at the moment. In the window, you can see a message showing Getting ready to create a backup and the Active users link. Click the link to view a list of connections to the application. The application attempts to terminate all user sessions.
  • After all user sessions are terminated, the application locks out and runs backup.
  • If the application cannot terminate all user sessions, it offers you to perform a backup while users are connected to the application. You can agree with the application offer if administrator is sure that connected users do not enter data to the application (for example, a user forgot to exit the application at the end of the day). Otherwise user sessions should be terminated locally.
  • After the backup is successful completed, the application starts again. You can see a message about performed backup indicating the path to the backup file.
  • If the application cannot forcibly terminate sessions, and administrator does not take any actions, backup might not be performed. In this case, the application displays a warning that backup cannot be performed, and a list of sessions that cannot be terminated (for example, Designer).

Starting application if errors occurred during backup

  • Before backup, the application displays a message with recommendations for immediate start.
  • To sign in to the infobase if backup is not performed, start 1C:Enterprise with parameters: ENTERPRISE /F <path to the infobase> /C AllowUserAuthorization /UC Backup.

Designed to restore data from the backup.

To access it, click Restore from backup in the Administration - Service - Backup and restoring section.

Data can be restored from a previously created backup file at any time. Please note that all data in the recovered infobase will be relevant for the time the backup is created. The recovered infobase will not contain data entered into the application after creating a backup.

Data recovery

  • By clicking Select, specify a path to the backup file from which the infobase will be recovered (the field is required).
  • When restoring data in this wizard, you can only use zip archives that contain one file - the 1cv8.cd infobase file . Archives generated by the backup wizard are better.
  • The application displays a warning that there are other users besides you working with the infobase at the moment. Click the link to View list of active sessions.
  • Click Restore data. If there are active sessions, disconnection mechanism will be started to disconnect the sessions from the infobase. Then the application will restore data.

Logging on to the infobase in case of data recovery failure

  • Important: To sign in to the infobase if data recovery is not performed, start 1C:Enterprise with parameters: ENTERPRISE /F <path to the infobase> /C AllowUserAuthorization /UC Backup.

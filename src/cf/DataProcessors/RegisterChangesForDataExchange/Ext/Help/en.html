This list is used to record application data changes. All data changes must be recorded before the data is sent to other application for synchronization purposes.

To open the list, click the button or execute the More - Sent data content command from the Data synchronization list.

All message numbers are recorded. To view or edit a message, click the link displaying its number.

The list consists of two sections:

  • The left section includes two columns:
    • Configuration object - this column lists applied objects (dictionaries, documents, etc) as a tree;
    • Number - the number of recorded items in these objects that were changed (or created) or were not exported;
  • Content of the right section varies depending on the object currently selected in the left section:
    • If multiple objects are selected, the Details column contains detailed statistics of the selected objects.
    • If a single object is selected, the right section is divided into two columns:
      • Object description (for example, Currency) - this column displays all list items that were changed but not yet exported;
      • Sent No - number of a sent message containing the changed list items (this column displays a message number or Has not exported message).

Changes filter

  • In the Display field, select the required value from the list:
    • All
    • Only exported;
    • Only not exported.

Recording changes, canceling recording

  • All changed and created objects are recorded automatically. To record an object manually, select the object in the tree by using its check box, and click Record changes:
    • - all objects of selected types - you can select the object itself and all objects subordinate to it;
    • - objects of selected types with auto-recording;
    • - single object (create the object, or select an object from list);
    • - filtered objects (set a filter and select the required items);
    • - query results.
  • To cancel sending all data prepared for other applications, select the object tree root (infobase) and click Cancel recording - All objects of selected types. In this case, the settings are identical to the above.

Settings

  • To set up data change recording parameters, click More - Setup. This allows you to set up auto-recording parameters, assign message number for recording purposes, or specify path to the query console directory.
  • Use check boxes to enable Autorecord of the following items:
    • Register records with documents;
    • Sequences with documents.
  • You can also Set message number on registration with radio buttons:
    • Register as unsent;
    • Use number of exchange node message.
  • To use the Query console, set the radio buttons to In directory, and specify the path to the query console. If the query console is not required, set the radio buttons to Do not use.
  • Select the check box to Use the Standard subsystem library when registering changes of data migration restriction objects.
  • For the changes to enter into force, click Write.

Viewing export results for DIB

  • To view the export results, select the object with recorded changes in the left section. In the right section, a list of object items will be displayed. Select a required item and click the Show export results button, or use More - Standard export results.

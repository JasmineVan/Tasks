Use this panel to set security profiles of the infobase for the 1C:Enterprise server cluster. Security profiles control the way the application is using the resources, which are external to the 1C:Enterprise server cluster.

To manage security profiles, go to Administration - Support and Maintenance - Security.

Security profile usage mode

You can select security profile usage mode for an application via radio button:

  • Security profiles are not used for this infobase
    • The application does not notify administrator that external resources must be added to security profiles.
      • If security profiles are enabled in a server cluster and this option is selected, any attempt to access external resources not permitted for this cluster server will result in application error.
      • If the Full access to external reports and data processors check box is not selected in security profile, the application will not start.
  • Security profiles for this infobase are set up by administrator via cluster console
    • The application does not notify administrator that external resources must be added to security profiles
    • The administrator sets up permissions in security profiles via cluster console.
      • The list of permissions that must be added to a security profile is available to the administrator in a report that can be accessed by clicking the Required permissions hyperlink on the panel.
      • The Safe mode profile check box must be selected for the infobase profile.
      • In the Infobase security profile field, fill in the name of security profile used for this infobase.
      • If the Full access to external reports and data processors check box is cleared in a security profile, select the Can be used as security profile check box and specify the security profile name in the Infobase security profile field on this panel. Otherwise, the application will not start.
  • Option to automatically add permissions to security profiles when necessary
    • When you switch to this mode, a new security profile will be created (and assigned to infobase) that permits access to all external resources required for the application to operate correctly.
    • Permissions in security profiles are managed by the application.
    • Profile management functionality is accessible from the application.
    • If while editing any application settings you need to gain access to a new external resource, an administrator receives a dialog box with the list of permissions.
      • If an administrator agrees to grant permissions, they enter authentication parameters for a cluster and click Next.
      • Permissions in security profiles are granted automatically.

Required permissions

This command cannot be used if Profile security usage mode is set to Security profiles are not used for this infobase.

  • Click Required permissions to display a report listing external resources that must be permitted in security profiles for the application to operate correctly.

Infobase security profile

This field is read-only if Profile security usage mode is set to Option to automatically add permissions to security profiles when necessary.

If the Profile security usage mode is set to Security profiles for this infobase are set up by administrator via cluster console and the Full access to external reports and data processors check box is not selected in the security profile, select the Can be used as security profile check box for the security profile and specify the name of this security profile in this field. Otherwise the application will not be able to start.

  • In the Infobase security profile field, enter the name of security profile to be used (in the 1C:Enterprise server cluster).

Restoring security profile settings

This command can be used only when Profile security usage mode is set to Option to automatically add permissions to security profiles when necessary.

This command fully rebuilds the infobase security profiles considering external resources required to run the application with current settings. This can be useful, for example, after the infobase was migrated between server clusters.

  • Click Restore security profile settings.

Editing external data processors or reports

  • Click Open external data processor or report. You can select the Secure mode to add an external module.

The form is designed to detect and delete duplicate items in all application lists. It is intended to be used by application administrators.

When working in the application, Users can create duplicates by mistake. For example, a user cannot find an item due to incorrect search request, and the item is created once again. This might prevent accounting procedures from being kept correctly. Documents start referring to different instances of the item, which leads to incorrect postings and invalid balance calculations. To avoid this, any duplicate item must be detected and deleted.

The application includes functionality to find duplicates marking them for deletion, and replace references to duplicate items with references to the "original" item.

The Search and deletion of duplicates command is located in the Administration - Service - Data adjustment section.

Searching for duplicates

  • In the Search in field, Select one of the application lists.
  • In the Filter field, specify list item filter criteria. To do so, click the All items link (default value), and set Filter rules.
    • Select a Field to filter by, and a search value. In the Comparison type field, select a comparison operation, and then specify a Value for comparison. Click OK to go back to duplicate search form. The hyperlink will change to display the filter conditions you selected.
    • To clear the filter, click .
  • In the Compare field, you can see predefined duplicate search rules that are inserted automatically as a link. To add a new item to the rules, click the link, select check boxes for the desired rule items, and then click Select. After that, the link will change its value.
  • After the duplicate search rules are complete, click Find duplicates.
  • If no duplicates were found, you can refine your search criteria, and then click Find duplicates.

List of duplicates

  • A list of found duplicates contains two sections:
    • The left section contains the found items. The items are displayed as a tree. The list includes the following columns:
      • Description. List items are grouped by name.
      • Code.
      • Usage instances. Number of item usage instances.
    • The right section displays information on the selected items:
      • item group (per each item). The report on the number of duplicates found for the selected item.
      • items. Usage report (a list of application documents using the item, or message "Not used").
  • One of the items used by the application is automatically marked as Original.
  • To mark a different item as original, select it, and then click Set as original.
  • Duplicates in the list are marked with flags.
  • To view the selected item, click Open. You can also double-click the item.
  • You can control visibility of item groups in the list by clicking Expand duplicate group, or Collapse duplicate group.
  • Infobar at the bottom of the list displays the number of items and the number of found duplicates.
  • If you want to improve the list of found duplicates, you can refine search conditions, and then click Retry search.

Deleting duplicates

  • If you agree with all conditions displayed in the list infobar, click Delete duplicates. All duplicates will be marked for deletion, and replaced with the selected Original throughout the documents.
  • To exit the form without deleting any duplicates, click Close. Search conditions and a list of found duplicated are not saved.
  • To delete duplicates permanently, click Delete marked objects in the Administration - Service section.

Viewing references to duplicate list items

  • Click All usage locations to view the Usage locations report for duplicates used in application documents.
